# ワークフロー最適化例

## 1. 学術研究ワークフロー

### 1.1 系統的文献レビュー

**従来の非効率なフロー**:
```
検索 → 結果確認 → 絞り込み → 再検索 → 確認 → ダウンロード → 整理
（各ステップで大量データがコンテキストを消費）
```

**最適化フロー**:
```
┌─────────────────────────────────────────────────────────┐
│ Phase 1: 初期検索                                       │
├─────────────────────────────────────────────────────────┤
│ 入力: キーワード、期間、最低引用数                        │
│ 処理: 複数DBから一括検索                                 │
│ 保存: /home/claude/slr/{project}/search_results.csv     │
│ 報告: 「{N}件発見。DB別内訳: SS {a}件, GS {b}件」         │
└─────────────────────────────────────────────────────────┘
                           ↓ 「続けて」
┌─────────────────────────────────────────────────────────┐
│ Phase 2: スクリーニング                                  │
├─────────────────────────────────────────────────────────┤
│ 入力: スクリーニング基準                                 │
│ 処理: タイトル/アブストラクトによる絞り込み               │
│ 保存: /home/claude/slr/{project}/screened.csv           │
│ 報告: 「{M}件に絞り込み（除外{N-M}件）」                  │
└─────────────────────────────────────────────────────────┘
                           ↓ 「続けて」
┌─────────────────────────────────────────────────────────┐
│ Phase 3: 詳細分析                                        │
├─────────────────────────────────────────────────────────┤
│ 入力: 分析フレームワーク                                 │
│ 処理: 選択論文の詳細読解・コーディング                    │
│ 保存: /home/claude/slr/{project}/analysis.md            │
│ 報告: 「主要テーマ: {theme1}, {theme2}, {theme3}」       │
└─────────────────────────────────────────────────────────┘
                           ↓ 「続けて」
┌─────────────────────────────────────────────────────────┐
│ Phase 4: 統合・執筆                                      │
├─────────────────────────────────────────────────────────┤
│ 処理: 文献レビューセクション執筆                          │
│ 出力: アーティファクト（学術論文形式）                    │
└─────────────────────────────────────────────────────────┘
```

**効果**:
- コンテキスト消費: 約75%削減
- 各フェーズの結果が保存され、後から参照可能

                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 2: 財務・業績データ収集                            │

│ ソース: IR資料、SEC EDGAR、有報                          │

│ 報告: 「財務データ収集完了。データ欠損: {欠損項目}」      │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 3: 戦略・施策の分析                                │
├─────────────────────────────────────────────────────────┤
│ ソース: ニュース、プレスリリース、アナリストレポート       │
│ 分析: 主要戦略、差別化要因、成功要因                      │
│ 保存: /home/claude/cases/{企業名}/strategy.md           │
│ 報告: 「戦略分析完了。各社の特徴的戦略を抽出」            │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 4: クロスケース分析                                │
├─────────────────────────────────────────────────────────┤
│ 処理: パターン抽出、類似点・相違点の分析                  │
│ 保存: /home/claude/cases/cross_analysis.md              │

## 2. データ分析ワークフロー

### 2.1 データ収集・クレンジング

**最適化フロー**:
```
┌─────────────────────────────────────────────────────────┐
│ Phase 1: データソース特定                                │
├─────────────────────────────────────────────────────────┤
│ 処理: 利用可能なデータソースの調査・リスト化              │
│ 保存: /home/claude/data/{project}/sources.md            │
│ 報告: 「{N}個のソースを特定。カバレッジ: {%}」           │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 2: データ取得                                      │
├─────────────────────────────────────────────────────────┤
│ 処理: 各ソースからデータ取得                             │
│ 保存: /home/claude/data/{project}/raw/                  │
│ 報告: 「取得完了。成功{a}件、失敗{b}件」                  │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 3: クレンジング                                    │
├─────────────────────────────────────────────────────────┤
│ 処理: 欠損値処理、外れ値検出、形式統一                    │
│ 保存: /home/claude/data/{project}/cleaned/              │
│ 報告: 「クレンジング完了。除外{N}件、補完{M}件」          │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 4: 統合・検証                                      │
├─────────────────────────────────────────────────────────┤
│ 処理: データ統合、整合性検証                             │
│ 保存: /home/claude/data/{project}/final.csv             │
│ 報告: 「統合完了。最終レコード数: {N}、変数数: {M}」      │
└─────────────────────────────────────────────────────────┘
```

## 3. 文書作成ワークフロー

### 3.1 学術論文執筆

**最適化フロー**:
```
Phase 1: 素材収集 → 各素材をファイル保存
Phase 2: アウトライン作成 → outline.md
Phase 3: 各セクション執筆 → sections/
Phase 4: 統合・推敲 → アーティファクト
```

### 3.2 プレゼンテーション作成

**最適化フロー**:
```
Phase 1: コンテンツ収集 → content/
Phase 2: ストーリー構成 → storyboard.md
Phase 3: スライド生成 → アーティファクト（pptx）
```
