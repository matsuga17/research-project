# RecursiveThink - 再帰的思考エージェント協働実行システム

## 概要

RecursiveThinkは、6つの専門思考エージェントを自動起動し、多視点分析を協調実行する高度な思考支援システムです。単一視点の限界を超え、体系的分析、創造的探索、批判的評価、統合的総合、実践的戦略立案、引用検証の6つの専門性を動的に組み合わせることで、複雑な問題に対する深い洞察と実行可能な解決策を生み出します。

### 主な用途
- 複雑な資料や問題の多角的分析
- 理論構築と仮説検証
- 戦略立案と意思決定支援
- 学術研究の品質向上
- 思考バイアスの防止と批判的検討

### 開発優先度
最高（プロジェクトの核心機能）

### 削減可能時間
- 多視点分析の手動実行：2-3時間から30分に短縮
- 思考バイアスの手動チェック：1時間から15分に短縮
- 統合レポート作成：1-2時間から30分に短縮
- 1回あたり削減時間：約3-4時間

## コア機能

### 1. エージェント自動配置

問題のタイプを自動判定し、最適なエージェント組み合わせを選択します。

問題タイプの分類：
- 技術的問題：体系的分析者 + 実践的戦略家 + 批判的評価者
- 社会的問題：創造的探索者 + 統合的総合者 + 批判的評価者
- 歴史的問題：体系的分析者 + 統合的総合者 + 引用検証専門家
- 学際的問題：全エージェント起動（6名体制）

自動配置プロセス：
1. 入力資料の内容分析
2. 問題領域の特定（技術/社会/歴史/学際）
3. 必要専門性の判定
4. エージェント組み合わせの決定
5. 各エージェントへのコンテキスト配信

### 2. 段階的思考プロセスの実行

4段階の構造化された思考プロセスを自動実行します。

#### 第0段階：原文忠実分析と構造化

担当：体系的分析者 + 引用検証専門家

実行内容：
- 資料の完全な理解と構造化
- 主要論点の抽出と分類
- 事実と主張の分離
- 引用と出典の初期検証
- 分析の基盤となる情報マップ作成

成果物：
- 構造化された資料マップ
- 主要概念の定義リスト
- 引用元の初期リスト

#### 第1段階：批判的分析と論点抽出

担当：批判的評価者 + 創造的探索者

実行内容：
- 前提条件と暗黙の仮定の特定
- 論理的欠陥と矛盾の検出
- 代替的解釈の探索
- 未検討の視点の特定
- 深堀りすべき論点の抽出

成果物：
- 批判的検討レポート
- 深堀り論点リスト
- 代替仮説リスト

#### 第2段階：多角的調査と証拠収集

担当：実践的戦略家 + 体系的分析者 + 引用検証専門家

実行内容：
- 論点ごとの情報源探索
- 信頼性の高いエビデンス収集
- 複数視点からのデータ統合
- 引用の厳格な検証
- 情報の信頼性評価

成果物：
- エビデンスマトリックス
- 検証済み引用リスト
- 情報源の信頼性評価

#### 第3段階：統合的考察と実装提案

担当：統合的総合者 + 実践的戦略家 + 創造的探索者

実行内容：
- 多様な洞察の統合
- 一貫したフレームワークの構築
- 実践的応用の提案
- 実装計画の策定
- 残された課題の明示

成果物：
- 統合レポート（アーティファクト形式）
- 実装ロードマップ
- 研究課題リスト

### 3. 思考劣化防止メカニズム

確証バイアスや思考の固着を積極的に防ぎます。

#### 反論生成プロトコル

実行タイミング：各段階の完了時

手順：
1. 現在のコンセンサスの明示化
2. 意図的な反論の生成（批判的評価者）
3. 反論の妥当性評価
4. 分析への統合または棄却
5. 修正された分析の再検討

#### 多様性強制メカニズム

検出条件：
- 同じ結論が3回以上繰り返される
- 単一の視点からの分析が2段階連続
- 特定のエージェントの発言が70%を超える

是正措置：
- 発言の少ないエージェントに強制発言権付与
- 意図的な視点シフトの実施
- 創造的制約の導入（例：「この前提が間違っていたら？」）

#### 思考リセット手法

適用条件：
- 解決策が見つからない状態が20分以上継続
- エージェント間の議論が収束しない
- 分析が表層的レベルに留まっている

リセット手順：
1. 現在の思考パターンの明示的記録
2. 5分間の強制休止
3. 全く異なる視点からの再開（例：「この問題を子供に説明するなら？」）
4. 新たな洞察の獲得
5. 元の分析との統合

### 4. 統合レポート生成

すべての分析結果をアーティファクト形式で自動生成します。

レポート構成：
1. エグゼクティブサマリー：主要発見と推奨事項
2. 段階別分析結果：第0-3段階の詳細
3. 視点間の対立と収束：エージェント間の議論の可視化
4. メタ認知的評価：思考プロセス自体の品質評価
5. 引用と参考文献：東京大学スタイル準拠
6. 次のステップ：残された課題と推奨アクション

出力形式：Markdown形式のアーティファクト

## 6つの思考エージェント

### 1. 体系的分析者 (Systematic Analyzer)

特性：精密、詳細志向、体系的

役割：複雑な問題の分解と構造化分析

得意領域：
- データの系統的整理
- 論理構造の解明
- 方法論的厳密性の確保
- 因果関係の特定

### 2. 創造的探索者 (Creative Explorer)

特性：発散的思考、パターン接続型、革新志向

役割：新しい視点と非従来的アプローチの提供

得意領域：
- 型破りな発想
- 異分野の知見の接続
- 代替的枠組みの提案
- 隠れたパターンの発見

### 3. 批判的評価者 (Critical Evaluator)

特性：懐疑的、検証志向、反証主義

役割：弱点の特定と前提への挑戦

得意領域：
- 論理的欠陥の検出
- 反証可能性の評価
- バイアスの特定
- 代替説明の探索

### 4. 統合的総合者 (Integrative Synthesizer)

特性：全体論的、関係性重視、文脈志向

役割：多様な視点の統合と全体像の構築

得意領域：
- 異なる洞察の接続
- 一貫したフレームワーク構築
- 矛盾の解消
- 全体像の把握

### 5. 実践的戦略家 (Practical Strategist)

特性：実用的、結果志向、資源意識

役割：理論から実践への橋渡し

得意領域：
- 実装可能性の評価
- リソース配分の最適化
- リスク管理
- 具体的行動計画

### 6. 引用検証専門家 (Citation Validator)

特性：精確、検証志向、形式重視

役割：引用・参考文献の厳格な管理

得意領域：
- 引用形式の統一
- URL検証と代替ソース探索
- 情報源の信頼性評価
- 著作権遵守の確保

## 使用方法

### 基本的な起動コマンド

RecursiveThinkを起動するには、以下のフォーマットを使用します：

```
RecursiveThink起動

分析対象：[資料名または問題記述]
問題タイプ：[技術的/社会的/歴史的/学際的]
重点視点：[オプション：特に重視したい視点]
出力形式：[統合レポート/段階別詳細/簡易版]
制約条件：[時間制限、文字数制限など]
```

### 使用例1：学術論文の批判的検討

```
RecursiveThink起動

分析対象：Smith et al. (2024)「E-commerce Platform Strategies in Asia」
問題タイプ：学際的（経営学×情報システム×地域研究）
重点視点：方法論的妥当性、理論的貢献、実証的エビデンスの質
出力形式：統合レポート
```

### 使用例2：研究計画の策定

```
RecursiveThink起動

分析対象：アジア太平洋E-commerce市場における戦略パターン分析 研究計画
問題タイプ：技術的（研究デザイン）
重点視点：データ収集の実現可能性、分析手法の適切性、スケジュール
出力形式：段階別詳細 + 実装ロードマップ
```

## 段階的実行プロセス

### 自動遷移トリガー

各段階の完了は以下の条件で自動判定されます：

第0段階から第1段階：
- 資料の構造化マップが完成
- 主要概念が定義済み
- 初期引用リストが作成済み

第1段階から第2段階：
- 深堀り論点が3つ以上特定
- 代替仮説が複数提示
- 批判的検討が完了

第2段階から第3段階：
- 論点ごとのエビデンスが収集済み
- 引用が検証済み
- 情報源の信頼性評価が完了

第3段階から完了：
- 統合フレームワークが構築
- 実装提案が具体化
- 最終レポートが生成

## ベストプラクティス

### 効果的な使用のための推奨事項

1. 明確な目的設定
   - 分析の目的を明確に指定
   - 期待する成果物のイメージを共有
   - 制約条件（時間、文字数等）を事前に提示

2. 適切な問題タイプの選択
   - 技術的：方法論、実装、データ分析
   - 社会的：人間行動、制度、政策
   - 歴史的：経路依存性、発展過程、比較分析
   - 学際的：複数領域にまたがる複雑な問題

3. 段階的な深化の活用
   - 第0段階で基盤をしっかり構築
   - 第1段階で批判的検討を徹底
   - 第2段階で十分なエビデンス収集
   - 第3段階で実践的な統合

## トラブルシューティング

### よくある問題と解決策

問題1：エージェント間の議論が収束しない
- 原因：前提条件の共有不足
- 解決策：第0段階に戻り、基本概念を再定義

問題2：表層的な分析に留まる
- 原因：批判的評価者の発言が不足
- 解決策：強制的に反論生成プロトコルを起動

問題3：実装提案が非現実的
- 原因：実践的戦略家の参加不足
- 解決策：第3段階で実践的戦略家の比重を増加

問題4：引用の検証に時間がかかる
- 原因：リンク切れが多い
- 解決策：Internet Archiveの活用、DOI優先使用

問題5：思考が固着している
- 原因：確証バイアスの発生
- 解決策：思考リセット手法の即座実行

## 品質保証

### 自動品質チェック項目

すべての分析完了時に以下を自動チェック：

- 6エージェントすべてが少なくとも1回は発言
- 第0-3段階すべてが完了
- 引用がすべて検証済み
- 反論生成プロトコルが少なくとも2回実行
- 統合レポートがアーティファクト形式
- エビデンスの信頼性評価が完了
- 実装提案が具体的
- メタ認知的評価が含まれる

### 品質基準

最低基準（必須）：
- すべての段階の完了
- 基本的な多視点分析
- 検証済み引用

推奨基準：
- 深い批判的検討
- 創発的洞察の獲得
- 実践的な実装計画

卓越基準：
- パラダイムシフトの提案
- 学際的統合の達成
- 革新的フレームワークの構築

## 結論

RecursiveThinkは、単なる分析ツールではなく、人間とAIの協働による知的探究のプラットフォームです。6つの専門エージェントの対話を通じて、単一視点では到達できない深い洞察と創造的解決策を生み出します。

思考劣化防止メカニズムにより、確証バイアスや固着を積極的に防ぎ、常に新鮮で批判的な視点を維持します。段階的プロセスは、表層的な分析から深層的な理解へと確実に導きます。

このスキルを活用することで、研究の質と効率が大幅に向上し、複雑な問題に対する革新的なアプローチが可能になります。
