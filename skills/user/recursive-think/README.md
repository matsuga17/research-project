# RecursiveThink - 再帰的思考エージェント協働実行システム

## 概要

RecursiveThinkは、6つの専門思考エージェントが協働して多視点分析を実行する、高度な思考支援スキルです。複雑な問題や資料に対して、体系的分析、創造的探索、批判的評価、統合的総合、実践的戦略立案、引用検証という異なる専門性を組み合わせることで、単一視点では到達できない深い洞察と実行可能な解決策を生み出します。

### 主な特徴

- ✅ **6つの専門エージェントによる多視点協働**
- ✅ **4段階の構造化された思考プロセス**（第0-3段階）
- ✅ **思考劣化防止メカニズム**（確証バイアス・固着の防止）
- ✅ **自動段階遷移**（完了条件の自動判定）
- ✅ **厳格な引用管理**（東京大学スタイル準拠）
- ✅ **アーティファクト形式の統合レポート**

### 削減効果

- 多視点分析：2-3時間 → 30分
- 思考バイアスチェック：1時間 → 15分
- 統合レポート作成：1-2時間 → 30分
- **合計：約3-4時間/回の削減**

## 6つの思考エージェント（概要）

### 1. 体系的分析者
問題を分解し、構造化分析を実行

### 2. 創造的探索者
新しい視点と非従来的アプローチを提供

### 3. 批判的評価者

### 5. 実践的戦略家
理論を実践に橋渡し、実装計画を策定

## よくある質問

### Q1: どのくらい時間がかかりますか？
**A**: 問題の複雑さにより異なりますが：
- 簡単な分析：30分-1時間
- 標準的な分析：1-2時間
- 複雑な分析：2-4時間

従来の手動分析と比較して約70%の時間削減が期待できます。

### Q2: どのような問題に適していますか？
**A**: 以下のような問題に特に有効です：
- 多様な視点が必要な複雑な問題
- 批判的検討が必要な分析
- 理論構築や仮説生成
- 実装計画を含む戦略立案

単純な情報収集や定型的なタスクには過剰です。

### Q3: カスタマイズは可能ですか？
**A**: はい、以下のカスタマイズが可能です：
- エージェントの組み合わせ調整
- 段階の追加・省略
- 専門エージェントの追加
- 出力形式の変更

詳細は[SKILL.md](SKILL.md)の「高度な活用テクニック」を参照。

### Q4: 他のスキルと連携できますか？
**A**: はい、以下のスキルと連携可能：
- **Tokyo-Cite**：引用管理の自動化
- **TrustGuard**：情報信頼性の強化
- **StratAnalyzer**：企業分析の効率化
- **ProjectFlow**：プロジェクト進捗管理

### Q5: 品質はどのように保証されますか？
**A**: 複数のメカニズムで品質を保証：
- 6エージェントの相互チェック
- 思考劣化防止メカニズム
- 段階完了条件の自動検証
- 最終品質チェックリスト

詳細は[SKILL.md](SKILL.md)の「品質保証」セクションを参照。

## トラブルシューティング

### 問題：分析が表層的に留まる
**解決策**：
1. 批判的評価者の発言を増やす
2. 反論生成プロトコルを強制実行
3. 第1段階に戻って論点を再抽出

### 問題：エージェント間の議論が収束しない
**解決策**：
1. 第0段階に戻り、基本概念を再定義
2. 統合的総合者の比重を増加
3. 明示的な統合基準を設定

### 問題：思考が固着している
**解決策**：
1. 思考リセット手法を即座実行
2. 創造的制約を導入
3. 全く異なる視点から再開

**詳細は[SKILL.md](SKILL.md)の「トラブルシューティング」を参照**
