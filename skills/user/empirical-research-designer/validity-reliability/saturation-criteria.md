# データ飽和判定基準：Data Saturation Criteria

## 概要

データ飽和（Data Saturation）は、質的研究において**新たなデータが新たな洞察をもたらさなくなった状態**を指す。これは、データ収集を終了する**理論的根拠**を提供し、研究の完全性（completeness）を示す重要な基準である。

Glaser & Strauss (1967)のグラウンデッド・セオリー研究で提唱され、現在では質的研究全般における**データ十分性**（data adequacy）の黄金律として広く認識されている。

---

## I. 理論的基盤

### 1.1 データ飽和の定義

**中核的定義**

データ飽和は、以下の3条件が満たされた状態である：

**条件1: 新情報の欠如（No New Information）**
- 追加のデータ収集（インタビュー、観察等）が、既存のテーマ・カテゴリーを超える新たな情報を生み出さない

**条件2: 理論的発展の停止（No New Theoretical Insights）**
- 新たなデータが、既存の理論的理解を拡張・修正しない

**条件3: カテゴリーの密度（Category Density）**
- すべての主要カテゴリーが十分な証拠で「飽和」し、理論的説明力を持つ

### 1.2 なぜデータ飽和が重要か

**問題1: データ不足（Under-saturation）**

データ収集を早期に終了すると：
- 重要なテーマ・パターンの見落とし
- 理論的説明の浅薄化
- 研究の信頼性・妥当性の欠如

**問題2: データ過剰（Over-saturation）**

必要以上にデータを収集すると：
- 時間・資源の浪費
- 参加者への過度な負担
- 分析の複雑化（膨大なデータの管理困難）

**データ飽和の価値**

適切なタイミングでデータ収集を終了し、**効率性**（efficiency）と**十分性**（sufficiency）のバランスを実現する。

### 1.3 研究パラダイムによる飽和の違い

**グラウンデッド・セオリー**
- 最も厳格な飽和基準
- 理論的飽和（theoretical saturation）：新たなデータが理論を発展させない

**現象学的研究**
- 経験の本質が捉えられるまで
- 構造的記述が完成するまで

**エスノグラフィー**
- 文化的パターンが明確になるまで
- 参加者の「厚い記述」（thick description）が可能になるまで

**ケーススタディ**
- ケースの包括的理解が達成されるまで
- 主要な因果メカニズムが特定されるまで

## II. データ飽和の3類型

### 2.1 コード飽和（Code Saturation）

**定義**

新たなデータから**新たなコード**（概念、テーマ）が出現しなくなった状態。

**判定方法**

**手順1: 継続的比較**

各新データ（インタビュー、観察）を、既存のコードブックと比較。

**例：AI導入研究**

**インタビュー1-5**：「初期不安」「学習曲線」「効率化」「役割変化」等、15の新コード発見

**インタビュー6-10**：8の新コード発見（「部門間対立」「技術的問題」等）

**インタビュー11-15**：3の新コード発見（「予期せぬ効果」等）

**インタビュー16-20**：0の新コード（既存コードのみで説明可能）

→ **コード飽和達成**（インタビュー16以降）

**定量的指標**

**新コード出現率**
```
新コード率 = (新たなコード数) / (全コード数) × 100%

推奨基準：新コード率 < 5% で3連続 → コード飽和
```

**例**
- インタビュー16：1新コード / 35全コード = 2.9%
- インタビュー17：0新コード / 35全コード = 0%
- インタビュー18：1新コード / 36全コード = 2.8%

→ 3連続で5%未満 → コード飽和

**注意点**

コード飽和は**必要条件**だが**十分条件ではない**。コードが出現しなくても、理論的理解は不十分な可能性がある。

### 2.2 意味飽和（Meaning Saturation）

**定義**

新たなデータが、既存のテーマ・カテゴリーに**新たな意味・次元**を追加しなくなった状態。

**コード飽和との違い**

- **コード飽和**：新たな「名前」（ラベル）が出現しない
- **意味飽和**：既存のコードに、新たな「意味」「ニュアンス」が追加されない

**例：「初期不安」というコード**

**インタビュー1-5**：「AI導入が不安」という基本的意味

**インタビュー6-10**：「不安」の多様な側面を発見
- 技術的不安（使いこなせるか）
- 雇用不安（仕事を奪われるか）
- 社会的不安（周囲に取り残されるか）

**インタビュー11-15**：さらなる次元を発見
- 時間的次元（初期 vs 中期の不安の変化）
- 強度的次元（軽度の懸念 vs 深刻な恐怖）

**インタビュー16-20**：これ以上の新たな意味が出現しない

→ **意味飽和達成**（インタビュー16以降）

**判定方法**

**チェックリスト方式**

各主要テーマについて、以下を確認：
- [ ] テーマの定義が明確か
- [ ] テーマの次元（時間、強度、種類等）が特定されたか
- [ ] テーマの境界（含まれるもの、含まれないもの）が明確か
- [ ] テーマ間の関係が理解されたか
- [ ] 新たなデータが上記のいずれも追加しないか

すべて「はい」→ 意味飽和達成

### 2.3 理論的飽和（Theoretical Saturation）

**定義**

新たなデータが、研究の**理論的枠組み**（因果関係、プロセスモデル、概念的枠組み）を発展・修正しなくなった状態。

**最も厳格な飽和基準**

グラウンデッド・セオリー研究で求められる最高水準の飽和。

**判定方法**

**基準1: 理論的モデルの完成**

研究の中核的問い（RQ）に答える理論的モデルが構築され、すべての主要要素（変数、関係、プロセス）が特定された。

**例：AI導入プロセスモデル**

**初期モデル（インタビュー1-10）**
```
AI導入決定 → 導入実施 → 結果評価
```

**修正モデル（インタビュー11-15）**
```
AI導入決定 → 組織準備 → 導入実施 → 適応期 → 定着期 → 結果評価
```

**最終モデル（インタビュー16-20）**

新たなデータも上記モデルで説明可能、モデル修正不要

→ **理論的飽和達成**

**基準2: カテゴリー間関係の明確化**

すべての主要カテゴリーが、他のカテゴリーとの関係（因果、相互作用、条件）において十分に説明された。

**例**

**関係1**：「経営層コミットメント」→「導入成功」（正の因果関係）
**関係2**：「現場抵抗」→「導入遅延」（負の因果関係）
**関係3**：「技術的問題」×「組織準備」→「適応期の長さ」（交互作用）

新たなデータが、新たな関係を示さない → 理論的飽和

**基準3: ネガティブケースの不在**

理論的モデルに矛盾する事例（ネガティブケース）が出現しない、または既存の理論的説明で対処可能。

**例**

**予測**：「経営層コミットメント高 → 導入成功」

**ネガティブケース**：Company Xは経営層コミットメント高だが失敗

**理論的説明**：Company Xは「技術的問題」が深刻で、コミットメントだけでは不十分だった

→ 理論を修正：「経営層コミットメント × 技術的準備 → 導入成功」

新たなネガティブケースが出現せず、理論が頑健 → 理論的飽和

## III. 実践ガイドライン

### 3.1 飽和の事前計画

**Step 1: 飽和タイプの選択**

研究目的に応じて、目指すべき飽和レベルを決定。

| 研究タイプ | 推奨飽和レベル |
|----------|-------------|
| 探索的研究 | コード飽和 |
| 記述的研究 | 意味飽和 |
| 理論構築研究（グラウンデッド・セオリー） | 理論的飽和 |
| ケーススタディ | 意味飽和 + 部分的理論的飽和 |

**Step 2: 初期サンプルサイズの設定**

**推奨範囲**（先行研究のメタ分析に基づく）

| 研究タイプ | 最小サンプル | 推奨サンプル | 最大サンプル |
|----------|------------|------------|------------|
| 現象学 | 6-10 | 10-15 | 20 |
| グラウンデッド・セオリー | 20-30 | 30-50 | 60 |
| エスノグラフィー | 30-50 | 50-100 | 制限なし |
| ケーススタディ | 4-10ケース | 6-12ケース | 15ケース |

**重要**：これらは目安であり、実際の飽和は**データの豊饒性**に依存する。

**Step 3: 段階的データ収集計画**

**Phase 1: 初期収集**（推奨サンプルの60%）

**Phase 2: 飽和評価**（継続的評価開始）

**Phase 3: 追加収集**（必要に応じて、推奨サンプルの100-120%まで）

### 3.2 継続的飽和モニタリング

**頻度**：3-5件の新データごとに飽和評価を実施

**評価シート例**

| 評価項目 | インタビュー11-15 | インタビュー16-20 | 判定 |
|---------|------------------|------------------|------|
| 新コード出現 | 3コード（6%） | 0コード（0%） | 飽和傾向 |
| 既存コードへの新意味追加 | 5コード | 1コード | 飽和傾向 |
| 理論的モデル修正の必要性 | なし | なし | 飽和 |
| ネガティブケース | 0件 | 0件 | 飽和 |
| **総合判定** | **継続** | **飽和達成** | **終了可** |

### 3.3 飽和判定の具体的手続き

**手続き1: コード飽和の確認**

**方法A: 新コード出現率**
```
新コード率 < 5% が3連続 → コード飽和
```

**方法B: コーディング飽和曲線**

横軸：データ数（インタビュー数）、縦軸：累積コード数

曲線が**プラトー**（plateau、平坦化）に達した点 → コード飽和

**例**
```
インタビュー1-5：  15コード（累積15）
インタビュー6-10： 8コード（累積23）
インタビュー11-15：3コード（累積26）
インタビュー16-20：1コード（累積27）
インタビュー21-25：0コード（累積27）
```

→ インタビュー16以降、曲線がプラトー → コード飽和

**手続き2: 意味飽和の確認**

**チェックリスト方式**（各主要テーマについて）

- [ ] 定義明確
- [ ] 次元特定
- [ ] 境界明確
- [ ] 関係理解
- [ ] 新情報なし

全テーマで全項目「はい」→ 意味飽和

**手続き3: 理論的飽和の確認**

**3ステップ検証**

**Step 1: モデル完成度**
- [ ] RQに答える理論的モデル構築済み
- [ ] すべての主要要素（変数、関係、プロセス）特定済み

**Step 2: カテゴリー間関係**
- [ ] すべての主要カテゴリーの関係（因果、相互作用、条件）明確

**Step 3: 頑健性**
- [ ] ネガティブケース不在、または理論で説明可能

全項目「はい」→ 理論的飽和

### 3.4 飽和達成後の追加確認

**飽和の誤判定を防ぐ**

飽和と判定した後も、**2-3件の追加データ**を収集し、判定の妥当性を確認することが推奨される。

**確認プロセス**

1. 飽和判定（例：インタビュー20で判定）
2. 追加データ収集（インタビュー21-23）
3. 再評価：新情報・新コード出現？
   - **なし** → 飽和確定
   - **あり** → 飽和未達、継続収集

## IV. サンプルサイズの理論的根拠

### 4.1 「何件で飽和？」の問いへの応答

**誤解**：「インタビュー〇件で飽和」という固定的基準

**正解**：飽和は**データの豊饒性**に依存し、固定的数値基準は存在しない

**影響要因**

**要因1: 研究目的の範囲**
- 狭い範囲（特定現象の深い理解）→ 少数で飽和（6-12件）
- 広い範囲（多様な経験の包括的理解）→ 多数必要（30-50件）

**要因2: データの豊饒性**
- 深いインタビュー（90-120分、構造化）→ 少数で飽和
- 浅いインタビュー（30分、非構造化）→ 多数必要

**要因3: サンプルの同質性**
- 同質的集団（単一企業、単一職種）→ 少数で飽和（10-15件）
- 異質的集団（多様な企業、職種）→ 多数必要（30-50件）

**要因4: 理論的感受性**
- 研究者の経験・知識が豊富 → 効率的飽和
- 初学者 → より多くのデータ必要

### 4.2 先行研究のベンチマーク

**現象学的研究**
- Morse (1994)：最低6件
- Creswell (1998)：5-25件
- Polkinghorne (2005)：5-25件

**グラウンデッド・セオリー**
- Morse (1994)：30-50件
- Creswell (1998)：20-30件
- Charmaz (2006)：25-30件（小規模プロジェクト）

**エスノグラフィー**
- Morse (1994)：30-50件
- Creswell (1998)：継続的参加観察（時間ベース、6-12ヶ月）

**ケーススタディ**
- Yin (2018)：4-10ケース
- Eisenhardt (1989)：4-10ケース

**重要な認識**

これらは**目安**であり、実際の飽和は研究の文脈に依存する。

## V. 高度な論点：飽和の批判と応答

### 5.1 批判1: 飽和は実証不可能

**批判**

「新情報なし」は証明不可能。次のデータが新情報をもたらす可能性は常に存在する。

**応答**

飽和は**絶対的確実性**ではなく、**合理的確信**（reasonable certainty）の問題である。

**実践的対応**
- 複数の飽和指標を組み合わせる（コード、意味、理論）
- 追加確認（飽和判定後2-3件追加）
- 透明な報告（判定根拠の明示）

### 5.2 批判2: 飽和は量的思考の押しつけ

**批判**

飽和概念は、質的研究に量的研究の「サンプルサイズ」論理を押しつけている。

**応答**

飽和は、量的研究の「統計的有意性」ではなく、質的研究固有の**理論的十分性**を示す。

**本質的違い**
- 量的：母集団への一般化可能性（representativeness）
- 質的：現象の理論的理解の深さ・包括性（comprehensiveness）

### 5.3 批判3: 飽和は早期終了の言い訳

**批判**

研究者が時間・資源不足で早期終了する際の「正当化」として飽和を利用する。

**応答**

この問題は認識しつつ、以下で対処：

**対処1: 飽和の透明な報告**
- 判定根拠を詳述（新コード率、意味飽和チェックリスト等）
- 判定プロセスの記録（いつ、どのように判定したか）

**対処2: 複数評価者による確認**
- 研究チーム、指導教員による独立評価

**対処3: 倫理的誠実性**
- 時間・資源制約を隠蔽せず、限界として透明に報告

## VI. 報告における提示

### 6.1 方法論セクションでの記述

**標準形式**

> **データ飽和**  
> 本研究では、Glaser & Strauss (1967)の理論的飽和の原則に従い、新たなデータが理論的枠組みを発展させなくなった時点でデータ収集を終了した。
> 
> 具体的には、各3-5インタビューごとに飽和評価を実施した。インタビュー16以降、新たなコードの出現率が5%未満となり（インタビュー16: 2.9%、17: 0%、18: 2.8%）、コード飽和の兆候が見られた。さらに、インタビュー16-20において、既存のテーマに新たな意味・次元が追加されず、理論的モデル（AI導入4段階プロセス）の修正も不要であった。
> 
> 飽和の妥当性を確認するため、インタビュー20で飽和判定後、追加2インタビューを実施したが、新たな情報・コード・理論的洞察は得られなかった。したがって、インタビュー22時点で理論的飽和が達成されたと判断し、データ収集を終了した（最終サンプル: n=22）。

### 6.2 結果セクションでの証拠提示

**飽和を示す証拠例**

> **コード飽和の証拠**  
> 図1に示すように、累積コード数は、インタビュー16以降でプラトーに達した。インタビュー1-10では平均2.3コード/インタビューの出現率であったが、インタビュー11-15では1.0コード/インタビュー、インタビュー16-22では0.14コード/インタビューに減少した。
> 
> **意味飽和の証拠**  
> 主要テーマ6つ（「初期不安」「学習曲線」「効率化」「役割変化」「部門間対立」「予期せぬ効果」）について、インタビュー16以降、新たな意味・次元は追加されなかった。例えば、「初期不安」テーマは、インタビュー1-15で7つの次元（技術的不安、雇用不安、社会的不安、時間的変化、強度的差異、対処戦略、緩和要因）が特定されたが、インタビュー16-22では新たな次元は出現しなかった。
> 
> **理論的飽和の証拠**  
> AI導入プロセスモデル（不安期→準備期→適応期→停滞期→定着期）は、インタビュー15で最終形に達し、インタビュー16-22で修正を要する事例は出現しなかった。また、モデルに矛盾するネガティブケースも検出されず、理論的頑健性が確認された。

## VIII. 実践例：AI導入研究におけるデータ飽和判定

### 研究概要
- テーマ：企業におけるAI導入プロセスの理論構築
- 方法論：グラウンデッド・セオリー
- 初期計画サンプル：20-30インタビュー

### 段階的飽和評価

**Phase 1: インタビュー1-10（初期収集）**

**結果**
- 新コード：23（累積23）
