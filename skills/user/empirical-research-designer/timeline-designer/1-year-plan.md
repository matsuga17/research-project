# 1年研究計画：1-Year Research Plan

## 概要

1年研究計画は、**混合法研究**、**多重ケーススタディ**、**博士論文エッセイ**など、中規模の実証研究プロジェクトに最適なタイムラインである。データ収集・分析・執筆の各フェーズに十分な時間を配分しながら、現実的な期間内での完了を可能にする。

---

## I. 適用場面と特徴

### 1.1 適用場面

**研究タイプ**
- 混合法研究（質的 → 量的、または量的 → 質的）
- 多重ケーススタディ（6-12ケース）
- 理論検証型サーベイ研究（n=200-500）
- 準実験デザイン研究
- 博士論文の個別エッセイ

**研究規模**
- データ収集：20-40インタビュー、またはサーベイ300-500件
- 分析期間：3-5ヶ月
- 執筆期間：2-3ヶ月

### 1.2 特徴と利点

**バランス型**
- データ収集・分析・執筆に均等な時間配分
- 深い分析と執筆品質の両立
- 複数の方法論の統合が可能

**柔軟性**
- 予期せぬ遅延への対応余地
- データ収集中の分析開始（並行作業）
- 段階的フィードバックの組み込み

**実行可能性**
- 1年という明確な目標設定
- 学術カレンダーとの適合（9月開始→8月完了等）
- 予算・リソースの計画可能性

## II. 標準タイムライン：4四半期構造

### Q1（Month 1-3）：準備・初期データ収集

#### Week 1-4（Month 1）：研究設計最終化・準備

**主要タスク**
- [ ] 文献レビュー完成・理論的枠組み確定
- [ ] 研究倫理審査申請・承認取得
- [ ] データ収集ツール最終化
  * インタビュープロトコル（質的研究）
  * サーベイ質問票（量的研究）
- [ ] サンプリング戦略実行開始
  * ケース企業コンタクト・アクセス交渉
  * サーベイ配布リスト作成
- [ ] パイロットテスト実施
  * パイロットインタビュー（2-3件）
  * パイロットサーベイ（30-50件）

**成果物**
- 倫理審査承認書
- 最終版データ収集ツール
- 確定したサンプルリスト

**留意点**
- 倫理審査は想定より時間がかかる（2-6週間）→ 早期申請
- パイロットテストで必ず修正点が見つかる→ 修正期間確保

#### Week 5-12（Month 2-3）：本格的データ収集開始

**質的研究の場合**
- [ ] インタビュー実施（10-15件、週2-3件ペース）
- [ ] 逐語録作成（各インタビュー後48時間以内）
- [ ] 予備的分析開始
  * メモライティング
  * 初期コーディング
- [ ] 文書収集・整理

**量的研究の場合**
- [ ] サーベイ配布（Week 5）
- [ ] リマインダー送信（Week 7, 9）
- [ ] データ収集継続・モニタリング
- [ ] 既存データベースからのデータ取得

**混合法研究の場合（質的先行型）**
- [ ] 質的フェーズ完全実施
- [ ] 質的データ初期分析
- [ ] 量的フェーズ設計開始

**Q1完了時の達成目標**
- データ収集40-50%完了
- 予備的分析の方向性確認
- 追加サンプリングの必要性評価

---

### Q2（Month 4-6）：データ収集完成・本格分析開始

#### Week 13-20（Month 4-5）：データ収集完成

**質的研究の場合**
- [ ] 残りインタビュー実施（10-15件）
- [ ] 理論的飽和の確認
- [ ] 追加サンプリング（必要な場合、2-3件）
- [ ] すべての逐語録作成完了
- [ ] 文書分析完成

**量的研究の場合**
- [ ] サーベイ回収締め切り（Week 14-16）
- [ ] データクリーニング
  * 欠損値処理
  * 外れ値検出・対応
  * 逆転項目の処理
- [ ] データ入力・コーディング完了

**混合法研究の場合**

**質的先行型（QUAL → quan）**
- [ ] 質的分析完成
- [ ] 量的フェーズ設計（質的結果に基づく）
  * 測定尺度開発
  * 仮説設定
- [ ] パイロットサーベイ実施（Week 18-20）

**量的先行型（QUAN → qual）**
- [ ] 量的分析完成
- [ ] 質的フェーズ設計
  * 興味深いケース選択
  * インタビュープロトコル開発
- [ ] インタビュー開始（Week 18-20）

#### Week 21-26（Month 6）：分析深化

**質的研究の場合**
- [ ] 本格的コーディング
  * オープンコーディング完成
  * 軸足コーディング
  * 選択的コーディング開始
- [ ] カテゴリー・テーマ抽出
- [ ] カテゴリー間関係性の検討
- [ ] メンバーチェック実施（該当する場合）

**量的研究の場合**
- [ ] 記述統計・相関分析
- [ ] 信頼性・妥当性検証
  * Cronbach's α
  * 確認的因子分析（CFA）
- [ ] 仮説検証分析
  * 回帰分析
  * 構造方程式モデリング（SEM）
- [ ] 頑健性チェック

**Q2完了時の達成目標**
- データ収集100%完了
- 分析60-70%完了
- 主要発見の輪郭が明確化
- 執筆の準備完了

---

### Q3（Month 7-9）：分析完成・執筆開始

#### Week 27-32（Month 7-8）：分析最終化

**質的研究の場合**
- [ ] 選択的コーディング完成
- [ ] 理論的統合
  * 中核カテゴリー特定
  * 理論モデル構築
- [ ] 分析の深化
  * 極端ケースの検討
  * 代替解釈の検証
- [ ] 分析結果の可視化
  * 概念図、プロセス図作成

**量的研究の場合**
- [ ] 主分析完成
- [ ] サブグループ分析
- [ ] 追加分析（探索的）
- [ ] 表・図の作成
  * 記述統計表
  * 相関行列
  * 回帰結果表
  * パス図（SEM）

**混合法研究の場合**
- [ ] 質的・量的結果の統合
- [ ] 統合マトリクス作成
- [ ] メタ推論（meta-inferences）の生成
- [ ] 統合解釈の深化

#### Week 33-39（Month 9）：執筆開始

**執筆戦略**：**非線形執筆**（結果→方法論→理論的枠組み→序論→考察→結論）

- [ ] **結果セクション**（Week 33-35）
  * 質的：テーマ・カテゴリーの記述
  * 量的：表・図を中心とした記述
  * 2,000-3,000字
  
- [ ] **方法論セクション**（Week 36-37）
  * 研究デザイン
  * サンプル
  * データ収集・分析手順
  * 1,500-2,500字
  
- [ ] **理論的枠組みセクション（ドラフト）**（Week 38-39）
  * 先行研究レビュー
  * 理論的基盤
  * 仮説/RQ導出
  * 2,000-3,000字

**Q3完了時の達成目標**
- 分析100%完了
- 論文50-60%完成（結果・方法論・理論セクション）
- 主要な論証構造確立

---

### Q4（Month 10-12）：執筆完成・レビュー・最終化

#### Week 40-46（Month 10-11）：執筆完成

- [ ] **序論**（Week 40-41）
  * 研究背景・問題意識
  * 研究目的・貢献
  * 論文構成
  * 1,000-1,500字

- [ ] **考察セクション**（Week 42-44）
  * 発見の解釈
  * 理論的含意
  * 実践的含意
  * 2,500-4,000字

- [ ] **結論セクション**（Week 45）
  * 要約
  * 理論的・実践的貢献
  * 限界
  * 将来研究の方向性
  * 1,000-1,500字

- [ ] **参考文献・付録整理**（Week 46）
  * 引用の一貫性確認
  * 参考文献リスト完成
  * 付録（インタビュープロトコル、測定尺度等）

#### Week 47-52（Month 12）：レビュー・最終化

**Week 47-48：自己レビュー**
- [ ] 全体の論理構造確認
- [ ] セクション間の整合性チェック
- [ ] 引用の正確性確認
- [ ] 文章の流れ・読みやすさ改善

**Week 49-50：指導教員・共同研究者レビュー**
- [ ] ドラフト送付
- [ ] フィードバック受領
- [ ] 主要な修正実施

**Week 51：最終化**
- [ ] すべてのフィードバック反映
- [ ] 最終校正
- [ ] 体裁・書式の統一
- [ ] 投稿準備（該当する場合）

**Week 52：完成・提出**
- [ ] 最終版完成
- [ ] 提出/投稿

**Q4完了時の達成目標**
- 論文100%完成（10,000-15,000字）
- 査読・投稿準備完了
- または学位論文章として完成

## III. 混合法研究の特殊なタイムライン

### 3.1 説明的順次デザイン（QUAN → qual）

```
Q1（Month 1-3）：量的フェーズ
- サーベイ設計・配布・回収
- データクリーニング
- 予備的分析

Q2（Month 4-6）：量的分析・質的フェーズ設計
- 量的分析完成
- 興味深いケース特定
- インタビュー実施（10-20件）

Q3（Month 7-9）：質的分析・統合
- 質的分析
- 量的・質的結果の統合
- 統合解釈

Q4（Month 10-12）：執筆
- 全セクション執筆
- レビュー・最終化
```

**特徴**
- 量的結果が質的フェーズの方向性を決定
- 統計的に有意な結果のメカニズム解明
- 予期せぬ結果の深層理解

### 3.2 探索的順次デザイン（QUAL → quan）

```
Q1（Month 1-3）：質的フェーズ
- インタビュー実施（15-25件）
- 予備的分析
- テーマ・概念抽出

Q2（Month 4-6）：質的分析・量的フェーズ設計
- 質的分析完成
- 測定尺度開発（質的結果に基づく）
- サーベイ設計・パイロットテスト

Q3（Month 7-9）：量的フェーズ・統合
- サーベイ配布・回収
- 量的分析
- 質的・量的結果の統合

Q4（Month 10-12）：執筆
- 全セクション執筆
- レビュー・最終化
```

**特徴**
- 質的研究で理論・仮説生成
- 量的研究で一般化可能性検証
- 新規測定尺度の開発

### 3.3 収束デザイン（Concurrent QUAL + QUAN）

```
Q1（Month 1-3）：並行データ収集
- インタビュー（15-20件）
- サーベイ配布・回収
- 両方同時進行

Q2（Month 4-6）：並行分析
- 質的分析
- 量的分析
- 独立に実施

Q3（Month 7-9）：統合・執筆開始
- 結果の比較・対比
- 統合マトリクス作成
- 統合解釈

Q4（Month 10-12）：執筆完成
- 全セクション執筆
- レビュー・最終化
```

**特徴**
- 質的・量的が同等の重要性
- 多角的検証（triangulation）
- 時間効率的

## IV. リスク管理とバッファ戦略

### 4.1 典型的リスクと対応

| リスク | 発生時期 | 影響度 | 対応策 | 所要時間 |
|-------|---------|-------|--------|---------|
| 倫理審査遅延 | Q1 | 高 | 早期申請、予備審査活用 | +2-4週 |
| サンプルリクルートメント困難 | Q1-Q2 | 高 | 予備リスト2倍、紹介活用 | +2-6週 |
| サーベイ回収率低迷 | Q2 | 中 | リマインダー、謝礼増額 | +2-4週 |
| 分析期間超過 | Q2-Q3 | 中 | 並行作業、優先順位付け | +2-4週 |
| 指導教員レビュー遅延 | Q4 | 低 | 早期ドラフト提出 | +1-2週 |

### 4.2 バッファ配置戦略

**Q1-Q2（データ収集・分析）**
- バッファ：各四半期2-3週
- 柔軟な調整が可能な期間

**Q3（分析完成・執筆開始）**
- バッファ：2週
- 分析遅延の吸収

**Q4（執筆・レビュー）**
- バッファ：3-4週（Week 48, 51-52）
- レビューフィードバックへの対応

**合計バッファ：8-10週（全体の15-20%）**

### 4.3 進捗モニタリング

**月次チェックポイント**

| 月 | チェック項目 | 達成基準 |
|----|------------|---------|
| 1 | 倫理審査承認、ツール最終化 | ✓ |
| 2-3 | データ収集40-50% | ✓ |
| 4-5 | データ収集100% | ✓ |
| 6 | 分析60-70% | ✓ |
| 7-8 | 分析100% | ✓ |
| 9 | 論文50-60%完成 | ✓ |
| 10-11 | 論文90%完成 | ✓ |
| 12 | 論文100%完成 | ✓ |

**遅延対応の意思決定ルール**

- **2週遅延**：現行計画継続、バッファで吸収
- **4週遅延**：優先順位再検討、一部分析の簡素化
- **6週以上遅延**：計画の大幅見直し、または期間延長

## V. 実践例

### 実践例1：質的研究（多重ケーススタディ、8社）

**RQ**：日本企業のAI導入戦略の多様性とその成功要因

**Q1（Month 1-3）**
- 倫理審査承認（Week 2）
- ケース企業選定・アクセス交渉（Week 3-6）
- パイロットインタビュー（Week 4-5）
- 本格インタビュー開始（Week 7-12、12件）

**Q2（Month 4-6）**
- インタビュー継続（Week 13-20、16件追加、合計28件）
- 逐語録作成・予備的分析並行
- 文書収集・分析（Week 13-26）
- データ飽和確認（Week 24）

**Q3（Month 7-9）**
- 本格的コーディング（Week 27-32）
- テーマ・カテゴリー抽出（Week 33-35）
- 理論的統合（Week 36-38）
- メンバーチェック（Week 38）
- 執筆開始：結果・方法論（Week 33-39）

**Q4（Month 10-12）**
- 理論的枠組み・序論・考察執筆（Week 40-46）
- 指導教員レビュー（Week 49-50）
- 最終化（Week 51-52）

**成果物**：12,000字学術論文、8社詳細ケース記述

### 実践例2：量的研究（サーベイ研究、n=350）

**RQ**：リーダーシップスタイルと従業員エンゲージメントの関係

**Q1（Month 1-3）**
- 測定尺度確定（Week 1-2）
- パイロットサーベイ（Week 3-4、n=50）
- サーベイ修正（Week 5-6）
- サンプルリスト作成（Week 7-8）
- 本格サーベイ配布（Week 9）
- リマインダー送信（Week 11）

**Q2（Month 4-6）**
- データ収集継続・締め切り（Week 14）
- データクリーニング（Week 15-17）
- 記述統計・相関分析（Week 18-20）
- CFA、信頼性・妥当性検証（Week 21-24）
- 仮説検証分析開始（Week 25-26）

**Q3（Month 7-9）**
- 仮説検証分析完成（Week 27-30）
- 頑健性チェック（Week 31-32）
- 追加分析（媒介・調整効果）（Week 33-35）
- 表・図作成（Week 36-37）
- 執筆開始：結果・方法論（Week 33-39）

**Q4（Month 10-12）**
- 理論的枠組み・序論・考察執筆（Week 40-46）
- レビュー・最終化（Week 47-52）

**成果物**：10,000字学術論文、投稿準備完了

### 実践例3：混合法研究（説明的順次、QUAN→qual）

**RQ**：AI投資と企業業績の関係、および成功メカニズムの解明

**Q1（Month 1-3）：量的フェーズ**
- サーベイ設計・配布（Week 1-9）
- データ収集（Week 9-12）

**Q2（Month 4-6）：量的分析・質的フェーズ設計**
- データクリーニング・分析（Week 13-20）
- 高業績企業特定（Week 21）
- インタビュー設計・実施（Week 22-26、15件）

**Q3（Month 7-9）：質的分析・統合**
- 質的分析（Week 27-35）
- 量的・質的統合（Week 36-39）

**Q4（Month 10-12）：執筆**
- 全セクション執筆（Week 40-46）
- レビュー・最終化（Week 47-52）

**成果物**：15,000字混合法論文

- [ ] リスク特定・対応策準備

- [ ] 月次目標の達成状況確認
- [ ] 遅延の早期検知
- [ ] バッファ使用状況把握
- [ ] 必要に応じて計画調整

### 完了時
- [ ] すべてのタスク完了確認
- [ ] 成果物品質確認
- [ ] 次の研究への教訓抽出
