# 混合法研究ガイド

## 概要

混合法研究は、質的・量的アプローチを統合し、現象の包括的理解を目指す。本ガイドは、3つの主要デザイン（収束、説明的順次、探索的順次）の実装を支援する。

---

## 1. 混合法研究の理論的基盤

### 1.1 定義（Creswell & Plano Clark 2018）

質的・量的データの収集・分析と、両結果の統合を通じて研究課題に取り組む研究アプローチ。

### 1.2 混合法の正当化理由

**補完性（Complementarity）**
- 質的：深さ、文脈、メカニズム
- 量的：広さ、一般化可能性、パターン
- 統合：包括的理解

**三角測量（Triangulation）**
- 複数の方法で同一現象を検証
- 妥当性の向上

**発展（Development）**
- 一方の結果が他方の設計を導く
- 測定尺度開発、サンプル選択

**拡張（Expansion）**
- 研究の範囲・射程の拡大
- 異なる側面の探索

## 2. 収束デザイン（Convergent Design）

### 2.1 特徴

**並行データ収集**
- 質的・量的データを同時期に収集
- 独立に分析
- 同等の重要性（QUAL + QUAN）

**適用場面**
- 現象の包括的理解
- 異なる視点からの妥当性確認
- 限られた時間での研究

### 2.2 実装プロセス

**Phase 1: 並行データ収集**（同時期）
```
[質的フェーズ]          [量的フェーズ]
- インタビュー          - サーベイ
- 観察                  - アーカイブデータ
- 文書分析              - 実験

Timeline: 同時進行（3-6ヶ月）
```

**Phase 2: 独立分析**
- 質的：テーマ抽出、パターン発見
- 量的：統計分析、相関・回帰

**Phase 3: 結果の統合**

**統合戦略1: 並行比較**
```
テーマ/発見     質的結果     量的結果     統合解釈
────────────────────────────────────────
AI導入効果     効率向上     業績向上+15%  一致・補完
組織抵抗       強い懸念     中程度相関    部分一致
リーダーシップ 重要な要因   有意な媒介効果 相互補強
```

**統合戦略2: データ変換**
- 質的データの数値化（質→量）
- 量的データの質的解釈（量→質）

**統合戦略3: 合同表示**
- 統合表・図の作成
- 質的引用+量的指標の並記

### 2.3 品質基準

**統合の質**
- 一致（Convergence）：結果が収束
- 不一致（Divergence）：結果が矛盾→追加分析
- 拡張（Expansion）：新たな洞察

## 3. 説明的順次デザイン（Explanatory Sequential Design）

### 3.1 特徴

**QUAN → qual の順次実施**
- 量的データ収集・分析が先
- 質的フェーズで量的結果を説明
- 量的優先（QUAN > qual）

**適用場面**
- サーベイ結果の深い理解
- 統計的に有意な結果のメカニズム解明
- 予期せぬ結果の探索
- 外れ値ケースの調査

### 3.2 実装プロセス

**Phase 1: 量的フェーズ**（6-9ヶ月）
```
1. サーベイ設計・実施（n=300-500）
2. データ収集
3. 統計分析（記述統計、回帰、SEM等）
4. 主要発見の特定
5. 質的フェーズの焦点決定
```

**Phase 2: 移行期**（1-2ヶ月）
```
1. 説明が必要な結果の特定
   - 有意な関係
   - 予期せぬ発見
   - 媒介・調整効果
   
2. 質的サンプルの選択
   - 極端スコア（高/低パフォーマンス）
   - 代表的スコア（平均的ケース）
   - 外れ値ケース
   
3. 質的プロトコル開発
   - 量的結果に基づく質問設計
```

**Phase 3: 質的フェーズ**（3-4ヶ月）
```
1. インタビュー実施（15-25件）
2. 質的データ分析
3. メカニズムの解明
4. 量的結果の文脈的理解
```

**Phase 4: 統合・解釈**（1-2ヶ月）
```
1. 量的パターンの質的説明
2. 理論的メカニズムの構築
3. 統合的結論の導出
```

### 3.3 質的サンプル選択戦略

**参加者選択モデル（Participant Selection Model）**
- 量的結果に基づくサンプル選択
- 例：高業績群5社 vs 低業績群5社

**説明モデル（Explaining Results Model）**
- 有意な結果の説明に焦点
- 例：AI投資→業績向上のメカニズム解明

**外れ値説明モデル（Outlier Explanation Model）**
- 統計的外れ値の深層理解
- 例：AI投資高額だが業績低迷企業

## 4. 探索的順次デザイン（Exploratory Sequential Design）

### 4.1 特徴

**QUAL → quan の順次実施**
- 質的データ収集・分析が先
- 量的フェーズで質的結果を検証・一般化
- 質的優先（QUAL > quan）

**適用場面**
- 新規現象の理論構築後の検証
- 測定尺度の開発
- 介入・実験の設計
- 理論の量的検証

### 4.2 実装プロセス

**Phase 1: 質的フェーズ**（6-9ヶ月）
```
1. インタビュー・観察（20-30件）
2. グラウンデッド・セオリー分析
3. テーマ・概念の抽出
4. 初期理論の構築
5. 量的フェーズの焦点決定
```

**Phase 2: 移行期**（2-3ヶ月）
```
1. 測定尺度開発
   - 質的テーマから項目生成
   - 専門家レビュー
   - パイロットテスト
   
2. サーベイ設計
   - 質的知見に基づく質問構成
   - 仮説定式化
   
3. サンプリング計画
   - 母集団特定
   - サンプルサイズ決定
```

**Phase 3: 量的フェーズ**（3-6ヶ月）
```
1. サーベイ実施（n=200-500）
2. データ収集
3. 統計分析（EFA, CFA, SEM等）
4. 質的知見の検証
5. 一般化可能性の評価
```

**Phase 4: 統合・解釈**（1-2ヶ月）
```
1. 質的理論の量的支持評価
2. 尺度の妥当性・信頼性確認
3. 修正理論の提示
```

### 4.3 尺度開発プロセス（Churchill 1979適用）

**ステップ1: 概念の明確化**（質的フェーズ）
- 構成概念の定義
- 理論的根拠

**ステップ2: 項目生成**（移行期）
- 質的テーマから項目作成
- 項目プール（目標の2-3倍）

**ステップ3: 尺度精緻化**（量的フェーズ）
- EFA（探索的因子分析）
- 項目削除（因子負荷量 < 0.50）
- 信頼性評価（α ≥ 0.70）

**ステップ4: 妥当性評価**（量的フェーズ）
- CFA（確認的因子分析）
- 収束的・弁別的妥当性
- ノモロジカル妥当性

## 5. 統合の方法論

### 5.1 統合ポイント

**設計段階（Design Level）**
- 研究課題の統合的定式化
- 質的・量的の補完的役割

**方法段階（Methods Level）**
- サンプリング戦略の統合
- データ収集の相互参照

**解釈段階（Interpretation Level）**
- 結果の統合的解釈
- メタ推論（meta-inferences）

**報告段階（Reporting Level）**
- 統合的論文構成
- 質的・量的結果の織り交ぜ

### 5.2 統合技法

**合同表示（Joint Display）**
```
構成概念   質的テーマ        量的指標      統合解釈
──────────────────────────────────────────
組織柔軟性 "迅速な適応"     r=0.65***    強い一致
          "試行錯誤の容認"   (業績との相関) 
AI受容性   "懐疑的態度"     平均=3.2/7   中程度の抵抗
          "学習意欲は高い"   SD=1.8       二極化
```

**データ変換**
- 質的→量的：コード頻度、テーマ強度スコア
- 量的→質的：統計結果の解釈的記述

**メタ推論**
- 質的・量的結果の総合的解釈
- 相互補完的洞察
- 新規理論的含意

## 6. 品質基準（Creswell & Plano Clark 2018）

### 6.1 妥当性の脅威

**設計妥当性**
- 不適切な混合法デザイン選択
- 質的・量的の不均衡

**正当化妥当性**
- 混合法使用の理由不明確
- 単一方法で十分な場合

**サンプル統合妥当性**
- 質的・量的サンプルの不整合
- 統合困難な異質サンプル

**統合妥当性**
- 不十分な統合
- 結果の単なる並列報告

### 6.2 品質評価基準

**統合の質**
- 質的・量的結果の明確な統合
- メタ推論の提示
- 矛盾の扱い（不一致時の追加分析）

**実施の質**
- 各方法論の厳密性維持
- 質的：信頼性、転移可能性
- 量的：妥当性、信頼性

**報告の質**
- 統合的論文構成
- 合同表示の効果的使用
- 統合プロセスの透明性

## 7. タイムライン例

### 収束デザイン（9ヶ月）
```
Month 1-3: 並行データ収集
  - 質的：インタビュー15件
  - 量的：サーベイn=300

Month 4-6: 独立分析
  - 質的：コーディング、テーマ抽出
  - 量的：記述統計、回帰分析

Month 7-8: 統合分析
  - 並行比較
  - 合同表示作成

Month 9: 統合報告書作成
```

### 説明的順次（15ヶ月）
```
Month 1-8: 量的フェーズ（QUAN）
  - 設計、データ収集、分析

Month 9-10: 移行期
  - 質的サンプル選択
  - プロトコル開発

Month 11-13: 質的フェーズ（qual）
  - インタビュー、分析

Month 14-15: 統合・執筆
```

### 探索的順次（18ヶ月）
```
Month 1-8: 質的フェーズ（QUAL）
  - インタビュー、グラウンデッド・セオリー

Month 9-11: 移行期
  - 尺度開発
  - サーベイ設計

Month 12-16: 量的フェーズ（quan）
  - データ収集、統計分析

Month 17-18: 統合・執筆
```

## 8. 報告ガイドライン

### 論文構成（説明的順次の例）

**序論**
- 統合的研究課題
- 混合法の正当化

**方法論**
- 混合法デザインの説明
- 質的・量的方法の詳細
- 統合戦略の記述

**結果**
- **Phase 1**: 量的結果（統計表、図）
- **Phase 2**: 質的結果（テーマ、引用）
- **統合**: 合同表示、統合解釈

**考察**
- メタ推論
- 理論的含意
- 統合的結論
