## 引用形式に関する問題

### 問題1: 著者名が不明

**症状**: Webサイトやレポートに著者名が記載されていない

**解決策**:

#### ステップ1: 著者情報の徹底探索
```
確認箇所:
□ ページ最上部・最下部
□ About/著者情報ページ
□ プロフィールセクション
□ 執筆者一覧ページ
□ メタデータ（HTML source確認）
```

#### ステップ2: 組織名の使用
```
✅ 良い例:
（経済産業省, 2023.4.15, p.57）
（日本経済新聞, 2024.2.1）

❌ 悪い例:
（著者不明, 2023.4.15）← これは使用しない
```

#### ステップ3: 編集者名の使用（書籍の場合）
```
形式: （編者名（編）, YYYY.M.D, p.XX）
例: （佐藤（編）, 2024.3.10, p.25）
```

#### ステップ4: 最終手段
```
形式: 『ウェブサイト名』(n.d.)「ページタイトル」
例: 『マーケティングポータル』(n.d.)「PB戦略の基礎」
```

---

### 問題2: 発行日が不明

**症状**: 正確な発行日が記載されていない

**解決策**:

#### ステップ1: 日付情報の徹底探索
```
確認箇所:
□ 記事冒頭・末尾
□ 「公開日」「投稿日」「更新日」
□ URLのパス（/2024/04/15/のような形式）
□ メタデータ（HTML source確認）
□ PDF のプロパティ（作成日/更新日）
```

#### ステップ2: 最終更新日の使用
```
✅ Webページに「最終更新：2024.5.10」がある場合
形式: （著者名, 2024.5.10）
注記: 参考文献に「最終更新日」と明記
```

#### ステップ3: アクセス日のみ使用
```
✅ オンライン資料で発行日不明の場合
形式: （著者名/組織名, n.d.）
参考文献: [アクセス日: YYYY.MM.DD]のみ記載
```

#### ステップ4: 概算日付の使用（非推奨）
```
⚠️ やむを得ない場合のみ
形式: （著者名, c.2023）← "c."は circa（おおよそ）の略
注記: 参考文献に「推定発行年」と明記
```

---

### 問題3: ページ番号が不明

**症状**: オンライン資料やPDFでページ番号がない

**解決策**:

#### ステップ1: PDF の場合
```
✅ PDFビューアのページ番号使用
形式: （著者名, YYYY.M.D, PDF p.10）
注記: 「PDFファイルのページ番号」と明記
```

#### ステップ2: セクション番号の使用
```
✅ セクション/章番号がある場合
形式: （著者名, YYYY.M.D, 第3章）
形式: （著者名, YYYY.M.D, Section 2.1）
```

#### ステップ3: パラグラフ番号の使用
```
✅ パラグラフ番号がある場合
形式: （著者名, YYYY.M.D, para. 5）
```

#### ステップ4: ページ省略（オンライン資料）
```
✅ Webページなどページ概念がない場合
形式: （著者名, YYYY.M.D）← ページ番号省略
```

### 問題4: 複数著者の記載方法

**症状**: 著者が複数いる場合の形式が不明

**解決策**:

#### 2名の場合
```
本文引用: （田中・山田, 2023.4.15, p.42）
参考文献: 田中太郎・山田花子（2023）...
```

#### 3名以上の場合
```
本文引用: （田中ほか, 2023.4.15, p.42）
参考文献: 田中太郎・山田花子・佐藤一郎（2023）...

注意: 本文は「ほか」、参考文献はすべての著者名を記載
```

#### 英語論文の場合
```
2名:
本文引用: （Smith & Johnson, 2023.3.15, p.32）
参考文献: Smith, John & Johnson, Mary（2023）...

3名以上:
本文引用: （Smith et al., 2023.3.15, p.32）
参考文献: Smith, John; Johnson, Mary; Brown, David（2023）...
```

### 問題5: 同姓の著者が複数

**症状**: 同じ姓の異なる著者が複数存在

**解決策**:

#### フルネーム使用
```
本文引用:
（田中太郎, 2023.4.15, p.42）
（田中花子, 2024.2.10, p.15）

参考文献:
田中太郎（2023）...
田中花子（2024）...
```

#### イニシャル追加（英語）
```
本文引用:
（Smith, J., 2023.3.15, p.25）
（Smith, M., 2024.1.10, p.30）

参考文献:
Smith, J.（2023）...
Smith, M.（2024）...
```

## URL検証に関する問題

### 問題6: URLが接続不能（404 Error）

**症状**: 引用したいURLにアクセスできない

**解決策**:

#### ステップ1: URL の再確認
```
チェック項目:
□ スペルミス確認
□ HTTPとHTTPSの違い確認
□ 末尾スラッシュの有無確認
□ 特殊文字のエンコーディング確認
```

#### ステップ2: Internet Archive検索
```
1. https://web.archive.org/ にアクセス
2. 元URLを入力して検索
3. 過去のスナップショット確認
4. 最新版のURLを取得

記載形式:
元URL [Internet Archive: アーカイブURL] [アクセス日: YYYY.MM.DD]
```

#### ステップ3: Google検索で代替URL探索
```
検索クエリ例:
"論文タイトル" "著者名"
site:ドメイン名 "キーワード"

発見した場合:
元URL [移転先: 新URL] [アクセス日: YYYY.MM.DD]
```

#### ステップ4: DOI検索（学術論文）
```
1. https://www.crossref.org/ で検索
2. 論文タイトル/著者名で検索
3. DOI発見時:

形式: https://doi.org/XX.XXXX/XXXXXX
DOI優先記載: 元URLではなくDOIを記載
```

#### ステップ5: 著者・出版社への直接問い合わせ
```
メールテンプレート:
---
件名: 論文「XXX」のURL確認のお願い

拝啓
貴論文「XXX」（YYYY年M月D日発表）を引用させていただきたく、
現在のURLをご教示いただけますでしょうか。

元URL: https://...（接続不能）
---
```

#### ステップ6: やむを得ない場合
```
形式:
[元URL - YYYY.MM.DD時点で接続不能。Internet Archive含む代替URL検索実施済み]

⚠️ 注意: この情報源の使用を再検討することを推奨
可能であれば、代替情報源を探索
```

### 問題7: URLが非常に長い

**症状**: URLが200文字以上で読みづらい

**解決策**:

#### ステップ1: Permanent Link確認
```
確認箇所:
□ ページ内の「Permanent Link」ボタン
□ 「Share」メニュー内
□ DOI（学術論文）
□ 短縮URL（公式提供されている場合）
```

#### ステップ2: DOI使用（学術論文）
```
✅ DOIがある場合は常に優先
例: https://doi.org/10.1234/example.2024
→ 元URLよりはるかに短く、永続的
```

#### ステップ3: 改行して記載
```
✅ 長いURLは改行可能
例:
https://www.very-long-domain-name.example.com/
research/publications/2024/march/article-about-ecommerce-strategy
[アクセス日: 2025.11.22]
```

#### ステップ4: 短縮URLの作成（非推奨）
```
⚠️ 最終手段のみ
理由: 短縮URLは永続性が保証されない

使用する場合:
元URL [短縮: bit.ly/example] [アクセス日: YYYY.MM.DD]
```

### 問題8: PDFファイルのURL

**症状**: PDFへの直リンクが不明

**解決策**:

#### ステップ1: PDF直リンクの取得
```
方法:
1. PDFを右クリック → 「リンクのアドレスをコピー」
2. PDFを開いた状態でURLバーを確認
3. ダウンロードリンクのURL使用
```

#### ステップ2: PDFである旨を明記
```
形式:
著者名（発行年）『タイトル』出版社, ページ数.
https://example.com/report.pdf [PDF] [アクセス日: YYYY.MM.DD]
```

#### ステップ3: ランディングページURLの使用
```
✅ PDF直リンクが不安定な場合
PDFが掲載されているページのURLを使用

形式:
https://example.com/publications/reports（PDF入手可能）
[アクセス日: YYYY.MM.DD]
```

## 参考文献リストに関する問題

### 問題9: ソート順が不明

**症状**: 日本語と英語の文献が混在している

**解決策**:

#### ルール
```
1. 日本語文献を先に記載（五十音順）
2. その後、英語文献を記載（アルファベット順）
3. セクションヘッダーで区別（オプション）
```

#### 実例
```
【参考文献】

■ 日本語文献

経済産業省（2023）...
佐藤一郎（2024）...
田中太郎（2023）...
山田花子（2022）...

■ 英語文献

Brown, David（2023）...
Smith, John（2024）...
World Bank（2023）...
```

---

### 問題10: 同一著者の複数文献

**症状**: 同じ著者の論文が複数ある場合のソート

**解決策**:

#### ルール
```
1. 著者名でまとめる
2. 発行年順（古い→新しい）にソート
3. 同年の場合はa, b, cで区別
```

#### 実例
```
田中太郎（2022）「第1報」...
田中太郎（2023a）「第2報」...
田中太郎（2023b）「第3報」...
田中太郎（2024）「第4報」...

本文引用:
（田中, 2023a.4.15, p.42）
（田中, 2023b.6.20, p.55）
```

---

### 問題11: 文献タイプの判定

**症状**: どの文献タイプに分類すべきか不明

**解決策**:

#### 判定フローチャート
```
学術雑誌に掲載? → YES → 雑誌論文形式
↓ NO
書籍として出版? → YES → 書籍形式
↓ NO
政府・国際機関発行? → YES → 政府刊行物形式
↓ NO
学会で発表? → YES → 会議発表形式
↓ NO
企業IR資料? → YES → 企業IR形式
↓ NO
Webページ → ウェブサイト形式
```

#### 迷った場合
```
✅ 最も詳細な情報を提供できる形式を選択
✅ 情報源の性質に最も近い形式を使用
✅ 一貫性を保つ（同種の情報源は同じ形式で統一）
```

## 整合性チェックに関する問題

### 問題12: 整合性チェックでエラー

**症状**: Tokyo-Cite整合性チェックでエラーが多数検出

**解決策**:

#### エラータイプ別対処

**タイプ1: 本文引用が参考文献リストに不在**
```
❌ 本文: （鈴木, 2021.5.20, p.25）
   参考文献リスト: 鈴木の文献なし

対処:
1. 参考文献リストに追加
2. または、本文引用を削除（使用していない場合）
```

**タイプ2: 参考文献リストが本文で未引用**
```
⚠️ 参考文献: 高田良一（2020）...
   本文: 高田の引用なし

対処:
1. 参考文献リストから削除
2. または、本文に引用を追加
```

**タイプ3: 日付形式不統一**
```
❌ （田中, 2023, p.42）← YYYY形式のみ
❌ （山田, 2022/6/10, p.15）← スラッシュ使用

✅ （田中, 2023.1.1, p.42）← YYYY.M.D形式
✅ （山田, 2022.6.10, p.15）← YYYY.M.D形式

対処:
すべての引用をYYYY.M.D形式に統一
```

**タイプ4: ページ形式不統一**
```
❌ （田中, 2023.1.1, p42）← ピリオドなし
❌ （山田, 2022.6.10, page 15）← "page"使用

✅ （田中, 2023.1.1, p.42）← p.形式
✅ （山田, 2022.6.10, pp.112-125）← pp.形式（複数ページ）

対処:
p./pp.形式に統一
```

### 問題13: 重複引用の検出

**症状**: 1ソース1引用原則違反

**解決策**:

#### 検出例
```
❌ 本文:
「田中（2023.4.15, p.42）によれば...」
「さらに田中（2023.4.15, p.58）は...」
← 同一ソースから2か所引用（違反）
```

#### 対処法1: 引用の統合
```
✅ 修正後:
「田中（2023.4.15, pp.42, 58）によれば、...と指摘している。
さらに、...とも述べている。」
```

#### 対処法2: パラフレーズ
```
✅ 修正後:
「田中（2023.4.15, p.42）によれば...」← 直接引用
「田中はまた、...とも述べている」← パラフレーズ（引用符なし）
```

#### 対処法3: 最重要箇所のみ引用
```
✅ 修正後:
「田中（2023.4.15, p.42）によれば...」← 最重要部分のみ直接引用
他の部分はパラフレーズまたは削除
```

## 特殊ケース

### 問題14: SNS投稿の引用

**症状**: Twitter(X)、Facebook等のSNS投稿を引用したい

**解決策**:

#### Twitter(X)の場合
```
形式:
投稿者名[@アカウント名]（投稿年月日）「投稿内容（最初の20語）」Twitter
URL [アクセス日: YYYY.MM.DD]

例:
田中太郎[@tanaka_taro]（2024.5.10）「本日の市場動向について分析しました」Twitter
https://twitter.com/tanaka_taro/status/1234567890 [アクセス日: 2025.11.22]
```

#### Facebook の場合
```
形式:
投稿者名（投稿年月日）「投稿内容（最初の20語）」Facebook
URL [アクセス日: YYYY.MM.DD]
```

#### 注意点
```
⚠️ SNS投稿の学術的引用は慎重に
理由:
- 投稿が削除される可能性が高い
- 学術的信頼性が低い
- できるだけ正式な公表物を使用

推奨:
1. 正式な論文・レポートがあれば、それを優先
2. SNS投稿は補足的に使用
3. スクリーンショット保存を推奨
```

### 問題15: YouTube動画の引用

**症状**: YouTube動画を引用したい

**解決策**:

#### 形式
```
投稿者名（投稿年月日）「動画タイトル」YouTube動画, 再生時間
URL [アクセス日: YYYY.MM.DD]
```

```
形式: （投稿者名, YYYY.M.D, MM:SS）← タイムスタンプ

```

#### 注意点
```
⚠️ 学術的引用としては慎重に
推奨:
1. 公式チャンネルの動画を優先
2. 信頼性の高い組織の動画を選択
3. 可能であれば、同内容の論文・レポートを使用
```

## よくある質問（FAQ）

### Q1: 引用の長さ制限（25語）をどう数える？

**A**: 以下のルールで数えます

```
日本語の場合:
- 助詞・助動詞も1語として数える
- 句読点は語数に含めない

例:「これは/重要な/発見/である」→ 4語

英語の場合:
- スペースで区切られた単位を1語
- ハイフンでつながれた語は1語

例: "This is an important discovery" → 5語
```

---

### Q2: 同じ情報源を複数箇所で言及する場合は？

**A**: パラフレーズと直接引用を使い分けます

```
✅ 1回目: 直接引用（引用符使用）
「田中（2023.4.15, p.42）は、「PB戦略の成功には品質が鍵」と指摘している。」

✅ 2回目以降: パラフレーズ（引用符なし）
「田中は、価格だけでなく品質も重視すべきと述べている。」

注意: 1ソース1引用原則により、直接引用は1か所のみ
```

---

### Q3: Web ページが頻繁に更新される場合は？

**A**: アクセス日を明記し、Internet Archiveに登録します

```
推奨対応:
1. アクセス日を必ず記載
2. Internet Archive（https://web.archive.org/）に保存
3. 重要な情報はスクリーンショット保存
4. PDFとして保存（バックアップ）

記載例:
https://example.com/dynamic-page [アクセス日: 2025.11.22]
[Internet Archive: https://web.archive.org/web/20251122/...]
```

---

### Q4: 著者の所属機関を記載すべき？

**A**: 基本的に不要ですが、同姓同名の識別が必要な場合のみ記載

```
❌ 通常は不要:
田中太郎（2023）『...』

✅ 識別が必要な場合のみ:
田中太郎（東京大学）（2023）『...』
田中太郎（慶應義塾大学）（2024）『...』
```

---

### Q5: URLにアクセス制限がある場合は？

**A**: アクセス制限を明記します

```
形式:
URL [要購読] [アクセス日: YYYY.MM.DD]
URL [要ログイン] [アクセス日: YYYY.MM.DD]
URL [機関アクセス限定] [アクセス日: YYYY.MM.DD]

例:
https://www.academic-journal.example.com/article/12345 [要購読] [アクセス日: 2025.11.22]
```

### Q6: 引用が25語を超えてしまう場合は？

**A**: 以下のいずれかの方法で対処します

```
方法1: 重要部分のみ抽出
❌ 「これは非常に重要な発見であり、今後の研究に大きな影響を与えるものと考えられる」（27語）
✅ 「今後の研究に大きな影響を与えるもの」（14語）

方法2: パラフレーズに変更
✅ 著者は、この発見が今後の研究に重要な影響を与えると述べている（引用符なし）

方法3: 要約して言及
✅ 著者は、この分野における重要な発見を報告している（引用符なし）
```

---

### Q7: 古い文献でURLがない場合は？

**A**: URL欄は省略可能です

```
形式:
著者名（発行年）『書名』出版社, ページ数.

例:
Porter, Michael E.（1985）『Competitive Advantage』Free Press, p.42.

注意: 最近の文献は可能な限りURL記載を推奨
```

---

### Q8: プレプリント（査読前論文）の引用は？

**A**: プレプリントである旨を明記します

```
形式:
著者名（発表年月日）「論文タイトル」『プレプリントサーバー名』プレプリント.
URL [アクセス日: YYYY.MM.DD]

例:
田中太郎（2024.5.10）「AI時代のマーケティング」『arXiv』プレプリント.
https://arxiv.org/abs/2405.12345 [アクセス日: 2025.11.22]

注意: 可能であれば正式発表後の論文を優先使用
```

## エラーメッセージと対処法

### エラー1: "URL connection failed (404)"

**意味**: URLにアクセスできません

**対処**:
1. URLのスペル確認
2. Internet Archive検索
3. Google検索で代替URL探索
4. 問題7を参照

---

### エラー2: "Citation format mismatch"

**意味**: 引用形式が統一されていません

**対処**:
1. すべての引用を（著者名, YYYY.M.D, ページ）形式に統一
2. 問題12を参照

---

### エラー3: "Duplicate citation detected"

**意味**: 同一ソースから複数引用されています

**対処**:
1. 引用を1か所に統合
2. または、パラフレーズに変更
3. 問題13を参照

---

### エラー4: "Reference not cited in text"

**意味**: 参考文献リストに記載されているが、本文で引用されていません

**対処**:
1. 本文に引用を追加
2. または、参考文献リストから削除

---

### エラー5: "Citation not in reference list"

**意味**: 本文で引用されているが、参考文献リストにありません

**対処**:
1. 参考文献リストに追加
2. または、本文の引用を削除
