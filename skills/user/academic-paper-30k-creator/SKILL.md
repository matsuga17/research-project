# Academic Paper 30k Creator
# 30,000字学術論文高速作成システム v1.0

## エグゼクティブサマリー

本スキルは、30,000字規模の学術論文を**効率的**かつ**高品質**に作成するための統合的システムです。既存の論文作成プロセス（45-50分）を**30-35分**に短縮（35%削減）し、同時にPublication-ready水準の品質を保証します。

### 中核機能

1. **論文構造自動設計**：最適な8セクション構成と文字数自動配分
2. **セクション別執筆支援**：各セクションの必須要素チェックリスト、論理展開テンプレート
3. **引用管理統合**：tokyo-citeスキルとの完全統合
4. **品質保証チェック**：論理的整合性、セクション間接続、理論的貢献の自動検証
5. **段階的作成プロセス**：Phase 1（構造設計5分）→ Phase 2（各セクション執筆25分）→ Phase 3（統合・調整8分）

### 成果指標

- **時間削減**：45-50分 → 30-35分（35%削減）
- **品質向上**：論理構造の明確化、セクション間一貫性保証、引用品質向上
- **エラー率**：構造的欠陥ほぼ0%、引用ミスほぼ0%
- **適用範囲**：戦略経営、組織論、産業分析、経済学、社会科学

### 他スキルとの統合

本スキルは以下のスキルと統合して使用します：
- **tokyo-cite**：引用管理（本文引用自動生成、参考文献リスト作成、URL検証）
- **strategic-organizational-empirical**：理論的基盤（RBV、TCE、制度理論、測定理論）
- **academic-paper-creation**：基礎ガイド（論述形式、日本語書式、理論構築原則）

## 1. システム概要

### 1.1 設計思想

本スキルは、以下の3つの設計思想に基づきます：

**効率性（Efficiency）**
- 論文構造の自動設計による初期段階の時間削減
- セクション別テンプレートによる執筆速度向上
- 引用管理の自動化による煩雑作業の排除

**厳密性（Rigor）**
- 理論的整合性の自動検証
- 方法論的厳密性の確保
- 学術的品質基準への準拠

**実用性（Practicality）**
- 段階的プロセスによる確実な進行
- チェックリストによる漏れの防止
- 実例による具体的ガイダンス

### 1.2 システムアーキテクチャ

```
Input（研究テーマ + データ）
    ↓
[Phase 1] 構造設計（5分）
    - 重要度判定
    - 8セクション構成決定
    - 文字数自動配分
    ↓
[Phase 2] セクション別執筆（25分）
    - セクション1: 序論（3分）
    - セクション2: 先行研究レビュー（4分）
    - セクション3: 理論的枠組み（3分）
    - セクション4: 研究方法論（2分）
    - セクション5: 実証分析（8分）
    - セクション6: 理論構築（3分）
    - セクション7: 考察（2分）
    ↓
[Phase 3] 統合・品質保証（8分）
    - セクション統合
    - 論理的整合性検証
    - 引用同期
    - 最終チェック
    ↓
Output（Publication-ready論文）
```

## 2. 段階的作成プロセス

### Phase 1: 論文構造自動設計（5分）

#### ステップ1-1: 重要度判定（2分）

以下の4軸で評価（各25点満点、合計100点）：

**影響範囲**（25点）
- 業界・分野への波及効果
- 社会的インパクト
- 政策的含意

**革新度**（25点）
- 理論的新規性
- 方法論的独創性
- 既存研究への挑戦度

**実用性**（25点）
- 具体的応用可能性
- 実務的示唆の明確性
- 実装可能性

**複雑度**（25点）
- 多角的視点の必要性
- 因果関係の複雑性
- データ分析の高度性

**重要度分類**：
- **80-100点**：最高重要度 → 30,000-32,000字、全セクション詳細化
- **60-79点**：高重要度 → 28,000-30,000字、標準構成
- **40-59点**：中重要度 → 25,000-28,000字、要点重視

#### ステップ1-2: 8セクション構成決定（2分）

**標準8セクション構成**（30,000字の場合）：

1. **序論**（3,000字、10%）
   - 問題意識と研究の背景
   - 研究目的とリサーチクエスチョン
   - 研究の意義と論文構成

2. **先行研究レビュー**（4,000字、13%）
   - 既存研究の体系的整理
   - 理論的空白の特定
   - 本研究の位置づけ

3. **理論的枠組み**（3,500字、12%）
   - 分析に用いる理論的基盤
   - 概念の定義と操作化
   - 分析枠組みの提示

4. **研究方法論**（3,000字、10%）
   - 研究設計の説明
   - データ収集方法
   - 分析手法

5. **実証分析/事例分析**（9,500字、32%）
   - データの提示と記述統計
   - 詳細な事例の検討
   - 分析結果の整理

6. **理論構築**（4,000字、13%）
   - 仮説の検証
   - 新規理論/概念の提示
   - 既存理論との関係性

7. **考察**（3,500字、12%）
   - 学術的インプリケーション
   - 実務的インプリケーション
   - 研究の限界

8. **結論**（1,500字、5%）
   - 研究の要約
   - 理論的貢献の再確認
   - 今後の研究課題

**参考文献**（1,000字、3%）

#### ステップ1-3: 文字数自動配分調整（1分）

重要度に応じて各セクションの文字数を微調整：

**最高重要度（30,000-32,000字）**
- 実証分析: 9,500字 → 10,000字
- 理論構築: 4,000字 → 4,500字
- 考察: 3,500字 → 4,000字

**高重要度（28,000-30,000字）**
- 標準配分を使用

**中重要度（25,000-28,000字）**
- 実証分析: 9,500字 → 8,000字
- 理論構築: 4,000字 → 3,500字
- 先行研究: 4,000字 → 3,500字

**Phase 1 完了チェックリスト**：
- [ ] 重要度判定完了（スコア記録）
- [ ] 8セクション構成確定
- [ ] 各セクション文字数配分決定
- [ ] 総文字数目標設定（28,000-32,000字）
- [ ] 執筆時間見積もり完了

### Phase 2: セクション別執筆（25分）

#### セクション1: 序論（3分、3,000字）

**目的**：読者を研究に引き込み、研究の必要性を説得的に示す

**必須要素チェックリスト**：
- [ ] 現実の問題提起（具体的数値・事例）
- [ ] 問題の重要性説明（複数視点）
- [ ] 既存研究の限界指摘
- [ ] 研究目的の明確な宣言
- [ ] リサーチクエスチョン提示
- [ ] 研究の意義（学術的・実務的）
- [ ] 論文構成の簡潔な紹介

**論理展開テンプレート**：

```
【段落1】問題提起（300-400字）
　現実の現象や社会的課題から始める。
　具体的な数値や事例で問題の深刻さを示す。

【段落2-3】背景説明（600-800字）
　なぜこの問題が重要なのか。
　どのような社会的・経済的文脈があるのか。
　複数の視点から問題の重要性を論証。

【段落4】研究の空白（400-500字）
　既存研究では何が明らかになっていないのか。
　理論的・実証的な空白を簡潔に指摘。

【段落5】研究目的（300-400字）
　本研究が明らかにすることを明確に宣言。
　リサーチクエスチョンを提示。

【段落6】研究の意義（400-500字）
　学術的貢献と実務的貢献を述べる。

【段落7】論文構成（200-300字）
　各章の内容を簡潔に紹介。
```

**品質基準**：
- 問題提起の具体性（数値・事例必須）
- 研究空白の明確性
- RQの具体性と検証可能性
- 論理的流れの自然さ

**執筆時のヒント**：
- 第一段落で読者の注意を掴む
- 「なぜこの研究が必要か」を繰り返し強調
- RQは1-3個に限定（明確さ重視）
- 専門用語は必要最小限に

**関連ガイド**：section-guides/introduction-guide.md

#### セクション2: 先行研究レビュー（4分、4,000字）

**目的**：研究の理論的位置づけを明確にし、理論的空白を特定する

**必須要素チェックリスト**：
- [ ] 主要理論の体系的整理

- [ ] 理論的・実証的空白の特定

**品質基準**：
- 体系性：散発的レビューではなく、構造化された整理
- 批判性：単なる要約ではなく、批判的検討
- 網羅性：主要な研究潮流を漏れなくカバー
- 焦点性：本研究に関連する部分に焦点

**引用の目安**：
- 総引用数: 20-30件
- 理論的基盤: 8-12件
- 実証研究: 12-18件
- 各段落: 1-2件の引用

## 3. 引用管理統合（tokyo-cite）

### 3.1 本文引用の自動生成

**基本形式**：
```
（著者名, YYYY.M.D, p.XX）
（著者名, YYYY, p.XX）
```

**tokyo-citeとの連携**：
1. 文献情報入力
2. tokyo-cite形式自動変換
3. 本文への挿入

**詳細**：tokyo-citeスキルの全機能が利用可能
- URL検証
- 代替URL探索（Internet Archive）
- DOI優先使用
- アクセス日自動記録

### 3.2 参考文献リストの自動生成

**生成プロセス**：
1. 本文中の全引用を抽出
2. tokyo-cite形式で参考文献生成
3. 五十音順/アルファベット順ソート
4. URL検証実施
5. 重複削除

**文献タイプ別対応**：
- 書籍
- 雑誌論文
- ウェブサイト
- 政府刊行物
- 会議発表

### 3.3 引用整合性チェック

**自動検証項目**：
- 本文引用と参考文献リスト完全一致
- 重複引用検出（1ソース1引用原則）
- 引用長制限（直接引用25語以下）
- 未引用文献の警告
- 存在しない引用の警告

**詳細**：citation-manager/tokyo-style-integration.md

## 5. 日本語学術書式規則

### 5.1 段落書式

**字下げルール**：
```
【見出し】字下げなし
1. 序論

【本文段落】全角スペース1文字
　本研究の目的は...

【引用文】全体を1文字字下げ
　Porter（1980）は以下のように述べている。

　　競争戦略の本質は...
```

### 5.2 論述形式原則

**箇条書き排除ルール**：

✅ **許可される箇条書き**：
- 参考文献リスト
- 図表のキャプション
- チェックリスト（メタ文書として）

❌ **禁止される箇条書き**：
- 論文本文での箇条書き
- 研究方法の説明
- 分析結果の列挙
- 考察・提言の提示

### 5.3 文章スタイル

**である調（常体）の使用**：
- 学術論文では「である」「だ」で統一
- 「です・ます調」は使用しない
- 断定的な表現を基本

**客観的記述**：
- 主観的表現を避ける
- 根拠のある主張には適切な表現
- 確実な事実は断定的に

**数値の表記**：
- パーセント：「51.1パーセント」または「51.1%」で統一
- 年号：西暦使用（2024年）
- 金額：「1兆5,000億円」のように3桁区切り

## 6. よくある問題と解決策

### 問題1: セクション間の文字数バランスが崩れる

**解決策**：
1. Phase 1で決定した文字数配分を厳守
2. 執筆中に定期的に文字数確認
3. 超過セクションは要点を絞る
4. 不足セクションは分析を深化

### 問題2: 論理的整合性が取れない

**解決策**：
1. Phase 3のステップ3-2を徹底実施
2. RQを常に意識しながら執筆
3. 理論的枠組みを分析の指針とする
4. セクション統合前に各セクションを見直し

### 問題3: 引用が不足または過剰

**解決策**：
1. 主要な主張には必ず引用
2. 自明な事実には引用不要
3. 1ソース1引用原則を遵守
4. tokyo-citeの整合性チェック活用

### 問題4: 時間内に完成しない

**解決策**：
1. Phase 1で重要度を正確に判定
2. 重要度を1段階下げて文字数削減
3. 各セクションの執筆時間を厳守
4. 完璧主義を避け、まず完成を優先
