# Tokyo-Cite統合ガイド
# 30,000字論文作成システムにおける引用管理

## 概要

本ガイドは、Academic Paper 30k CreatorとTokyo-Citeスキルの完全統合プロセスを解説します。引用管理の自動化により、週3-8時間の作業時間削減とほぼ0%のエラー率を実現します。

---

## 1. 統合アーキテクチャ

### 1.1 システム連携図

```
[Phase 2: セクション執筆]
    ↓ 引用情報記録
[Tokyo-Cite: 情報収集]
    ↓ 形式変換
[Tokyo-Cite: 本文引用生成]
    ↓ 論文内挿入
[Phase 3: 統合・品質保証]
    ↓ 全引用抽出
[Tokyo-Cite: 参考文献生成]
    ↓ URL検証
[Tokyo-Cite: 整合性チェック]
    ↓ 最終確認
[Publication-ready論文]
```

### 1.2 統合のメリット

**効率化**：
- 引用作成時間: 2-3分/件 → 30秒/件（83-85%削減）
- 参考文献リスト: 30分 → 5分（83%削減）
- URL検証: 1-2分/件 → 10秒/件（92-95%削減）

**品質保証**：
- 引用ミス: ほぼ0%（手動: 5-10%）
- URL切れ: ほぼ0%（手動: 20-30%）
- 形式不統一: 0%（手動: 15-25%）

## 2. Phase 2での引用管理

### 2.1 執筆中の引用情報記録

セクション執筆時に、引用が必要な箇所で即座に情報を記録します。

**記録フォーマット**：
```
[Tokyo-Cite記録]
タイプ: [書籍/論文/ウェブ/報告書/会議]
著者: [著者名]
タイトル: [タイトル]
発行年月日: [YYYY.M.D または YYYY]
URL: [URL]
ページ: [p.XX or pp.XX-XX]
追加情報: [出版社/雑誌名/会議名等]
```

**実例**：
```
[Tokyo-Cite記録]
タイプ: 雑誌論文
著者: 山田花子
タイトル: プライベートブランドの新展開
発行年: 2022
雑誌名: マーケティングジャーナル
巻号: 15(2)
ページ: pp.112-125
URL: https://www.marketing-journal.jp/article/12345
```

### 2.2 本文引用の即座生成

記録した情報から、Tokyo-Cite形式の本文引用を即座に生成します。

**生成コマンド**：
```
Tokyo-Cite: 本文引用を生成
[上記の記録情報]
```

**出力例**：
```
本文引用: （山田, 2022, pp.112-125）

または

本文引用: （山田, 2022）
※ページ指定が不要な場合
```

### 2.3 執筆フローへの統合

**統合パターン1: 論述の裏付け**
```
執筆中の文章:
「プライベートブランドは近年、消費者の選択肢として
重要性を増している。」

↓ 引用追加

最終文章:
「プライベートブランドは近年、消費者の選択肢として
重要性を増している（山田, 2022, p.115）。」
```

**統合パターン2: 直接引用**
```
執筆中の文章:
「山田（2022）は、PBの成長要因を分析し、...」

↓ 直接引用追加（25語以下厳守）

最終文章:
「山田（2022, p.115）は、「価格競争力と品質向上が
PB成長の二大要因」と指摘している。」
```

### 2.4 引用記録の一元管理

執筆中の全引用情報を一箇所に集約します。

**管理ファイル例**：
```
# 引用記録リスト（論文タイトル）

## 序論
1. [Tokyo-Cite記録] 経済産業省（2023）...
2. [Tokyo-Cite記録] キーエンス（2024）...
3. [Tokyo-Cite記録] Porter（1980）...

## 先行研究レビュー
4. [Tokyo-Cite記録] Barney（1991）...
5. [Tokyo-Cite記録] Wernerfelt（1984）...
...

## 実証分析
...
```

## 3. Phase 3での参考文献生成

### 3.1 全引用の自動抽出

Phase 2で記録した全引用情報を自動抽出します。

**抽出コマンド**：
```
Tokyo-Cite: 引用リスト抽出
対象: [論文全文]
```

**出力例**：
```
抽出された引用（30件）:
1. 経済産業省（2023）
2. キーエンス（2024）
3. Porter（1980）
4. Barney（1991）
...
30. 佐藤（2024.3.15）
```

### 3.2 参考文献リストの自動生成

Tokyo-Cite形式で参考文献リストを生成します。

**生成コマンド**：
```
Tokyo-Cite: 参考文献リスト生成
入力: [引用記録リスト]
ソート: 自動（五十音順/アルファベット順）
```

**出力例**：
```
## 参考文献

経済産業省（2023）『令和5年版 小売業白書』経済産業省, p.57.
https://www.meti.go.jp/report/retail2023 [アクセス日: 2025.11.22]

キーエンス株式会社（2024）「2024年3月期決算説明資料」
https://www.keyence.co.jp/ir/library/statement/ [アクセス日: 2024.5.20]

山田花子（2022）「プライベートブランドの新展開」『マーケティングジャーナル』15(2), pp.112-125.
https://www.marketing-journal.jp/article/12345 [アクセス日: 2025.11.22]

Barney, J. B. (1991) "Firm Resources and Sustained Competitive Advantage," Journal of Management, 17(1), pp.99-120.

Porter, M. E. (1980) Competitive Strategy, Free Press, pp.1-396.
```

### 3.3 URL検証の自動実行

すべてのURLを自動検証し、接続不能なURLには代替URLを探索します。

**検証コマンド**：
```
Tokyo-Cite: URL一括検証
対象: [参考文献リスト全URL]
```

**出力例**：
```
URL検証結果:
✅ https://www.meti.go.jp/report/retail2023 (200 OK)
✅ https://www.keyence.co.jp/ir/library/statement/ (200 OK)
✅ https://www.marketing-journal.jp/article/12345 (200 OK)
❌ https://old-site.example.com/article/999 (404 Not Found)
   → Internet Archive発見: https://web.archive.org/...
```

**代替URL記載例**：
```
旧URL（接続不能）:
https://old-site.example.com/article/999 [Internet Archive: https://web.archive.org/web/20220315100000/https://old-site.example.com/article/999] [アクセス日: 2025.11.22]
```

## 4. 整合性チェックの自動化

### 4.1 本文引用と参考文献の完全一致

**チェックコマンド**：
```
Tokyo-Cite: 整合性チェック
本文引用リスト: [(経済産業省, 2023), (キーエンス, 2024), ...]
参考文献リスト: [経済産業省（2023）..., キーエンス（2024）..., ...]
```

**チェック項目**：
1. 本文で引用されたすべての文献が参考文献リストに存在
2. 参考文献リストのすべての文献が本文で引用されている
3. 著者名・発行年の完全一致
4. 形式の統一性

**出力例（問題なし）**：
```
✅ 本文引用と参考文献の一致: 確認完了（30件）
✅ 重複引用チェック: 問題なし
✅ 形式統一性: 確認完了
✅ すべての検証項目をクリア
```

**出力例（問題あり）**：
```
✅ 本文引用と参考文献の一致: 確認完了（28/30件）
⚠️  未引用文献: 2件
   - 佐藤一郎（2024）「小売業におけるAI活用」
     ※本文で引用されていません
   - 鈴木花子（2023）「デジタル変革の課題」
     ※本文で引用されていません

❌ 存在しない引用: 2件
   - （田中, 2021）※参考文献リストに不在
   - （木村, 2023）※参考文献リストに不在

推奨対応:
1. 未引用文献の削除または本文への引用追加
2. 存在しない引用の削除または参考文献への追加
```

### 4.2 重複引用チェック（1ソース1引用原則）

**チェックコマンド**：
```
Tokyo-Cite: 重複引用チェック
対象: [論文全文]
```

**出力例（問題なし）**：
```
✅ 1ソース1引用原則: 遵守
   すべての文献から1箇所のみ引用されています。
```

**出力例（問題あり）**：
```
❌ 重複引用検出: 3件
   - 山田（2022）: 2箇所で引用
     * p.115: 序論
     * pp.112-125: 先行研究レビュー
   - Porter（1980）: 3箇所で引用
     * p.35: 序論
     * pp.40-45: 先行研究レビュー
     * p.50: 理論的枠組み

推奨対応:
各文献から最も重要な1箇所のみ残し、他は削除またはパラフレーズ
```

### 4.3 引用長制限チェック（直接引用25語以下）

**チェックコマンド**：
```
Tokyo-Cite: 引用長チェック
対象: [論文全文の直接引用]
```

**出力例（問題なし）**：
```
✅ 引用長制限: すべての直接引用が25語以下
   直接引用数: 8件
   最長: 23語（山田, 2022）
```

**出力例（問題あり）**：
```
❌ 引用長超過: 2件
   - 引用1（Porter, 1980）: 32語
     「競争戦略の本質は、企業がいかにして持続的な
     競争優位を構築し、それを維持するかにある。
     そのためには...」
     → 推奨: パラフレーズまたは重要部分のみ抽出

   - 引用2（山田, 2022）: 28語
     「プライベートブランドの成長要因は、価格競争力と
     品質向上の両面から...」
     → 推奨: 「価格競争力と品質向上が成長の鍵」に短縮
```

## 5. 形式統一の自動化

### 5.1 日付形式の統一

**ルール**：
- 書籍・学術論文: YYYY形式（例: 2023）
- ウェブ・ニュース: YYYY.M.D形式（例: 2024.2.1）
- 会議発表: YYYY.M.D形式（例: 2024.3.15）

**統一コマンド**：
```
Tokyo-Cite: 日付形式統一
対象: [全引用]
```

**出力例**：
```
日付形式統一完了:
✅ 書籍: 15件 → YYYY形式
✅ 学術論文: 10件 → YYYY形式
✅ ウェブサイト: 3件 → YYYY.M.D形式
✅ 会議発表: 2件 → YYYY.M.D形式
```

### 5.2 著者名形式の統一

**ルール**：
- 日本語: 姓のみ（例: 田中）
- 英語: Last name, First initial（例: Porter, M. E.）

**統一コマンド**：
```
Tokyo-Cite: 著者名形式統一
対象: [全引用]
```

**出力例**：
```
著者名形式統一完了:
✅ 日本語著者: 20件 → 姓のみ
✅ 英語著者: 10件 → Last, F. I.形式
```

### 5.3 ページ表記の統一

**ルール**：
- 単一ページ: p.42
- 複数ページ: pp.42-45
- ページなし: 省略

**統一コマンド**：
```
Tokyo-Cite: ページ表記統一
対象: [全引用]
```

## 7. トラブルシューティング

### 問題1: 重複引用が多数検出される

**原因**：
同じ文献から複数箇所で引用している

**解決策**：
1. 最も重要な1箇所のみ残す
2. 他の箇所はパラフレーズに変更
3. どうしても複数必要な場合は、異なるページから引用

### 問題2: URL検証で多数のエラー

**原因**：
古い資料のURLが切れている

**解決策**：
1. Tokyo-Cite: 代替URL自動探索
2. Internet Archiveから取得
3. DOI使用（学術論文）
4. 公式サイトで代替ページ探索

### 問題3: 本文引用と参考文献が不一致

**原因**：
執筆中の追加・削除による不整合

**解決策**：
1. Tokyo-Cite: 整合性チェック実行
2. 未引用文献の削除
3. 存在しない引用の追加または削除

### 問題4: 直接引用が長すぎる

**原因**：
重要部分の抽出不足

**解決策**：
1. 引用の核心部分のみ抽出（10-15語目標）
2. 前後の文脈はパラフレーズ
3. 複数の短い引用に分割（ただし1ソース1引用原則注意）

## 8. ベストプラクティス

### 8.1 執筆段階のベストプラクティス

**実践1: 早期記録**
- 文献を読んだ直後に引用情報を記録
- 後で探す手間を完全に排除

**実践2: 完全情報**
- すべての必須項目を初回から記録
- 不完全な記録は後で問題化

**実践3: URL即時検証**
- 資料収集直後にURL接続テスト
- 切れたURLを早期発見

**実践4: DOI優先**
- 学術論文は常にDOI確認・使用
- 永続性の高いURLを優先

### 8.2 統合段階のベストプラクティス

**実践1: 段階的チェック**
- 整合性チェックを複数回実行
- 問題を早期発見・早期解決

**実践2: バックアップ**
- 引用記録リストをバックアップ
- Internet Archiveへの重要文献登録

**実践3: 最終確認**
- 提出前にすべてのURLを再検証
- 形式統一を再確認

## 9. 品質保証基準

### 9.1 必須基準

**引用形式**：
- [ ] すべての本文引用が（著者, YYYY または YYYY.M.D, p.XX）形式
- [ ] 直接引用は25語以下
- [ ] 1ソース1引用原則遵守

**参考文献リスト**：
- [ ] すべての文献が適切な形式
- [ ] URLすべて記載（接続確認済み）
- [ ] アクセス日すべて記載
- [ ] 五十音順/アルファベット順ソート完了

**整合性**：
- [ ] 本文引用と参考文献リスト完全一致
- [ ] 重複引用なし
- [ ] 未引用文献削除済み
- [ ] 存在しない引用なし

### 9.2 推奨基準

**URL品質**：
- [ ] DOI使用（学術論文）
- [ ] Permanent Link優先
- [ ] Internet Archive登録（重要文献）

**形式統一**：
- [ ] 日付形式統一（文献タイプ別）
- [ ] 著者名形式統一
- [ ] ページ形式統一
